<objective>
Implement the measurement-INDEPENDENT diagnostic logging core — Priorities 1-3 of
`09-RL-LOGGING-REQS.md` (D-81-03) — so the next run can see WHICH reward component is flat and
whether GSPO groups are collapsing. Aggregate per-group reward stats inside
`compute_rollout_advantages` BEFORE the uniform-group drop, thread them through the existing
`(data, advantages, meta)` return as extra `meta` keys, and emit them plus policy/optimizer health
(entropy the single most informative add) from `_log_step` every step.

Purpose: The flat-reward episode was undiagnosable because `rl_metrics.jsonl` logged only the
composite reward. `frac_groups_all_zero` — the primary kill/continue signal — was never logged.
This plan adds exactly the visibility that would have caught the collapse immediately, and is the
prerequisite that lets Plan 04's offline signal-check confirm a restored per-group gradient.

Output: extended `scripts/rl_rollouts.py` (pre-drop aggregation), extended `scripts/rl_train.py`
(_log_step + run_training_step threading), new `tests/test_rl_rollouts_reward_shape.py`.
</objective>
