## What Was Done

### Task 1: Probe Extension + OFFLINE GATE

Extended `scripts/_probe_rl_reward.py` in two parts:

1. **Wired `_fix_score_from_completion`** (Lever 1 Form A) into `probe_histogram`. Previously the probe used the old binary parse gate (`ok→rubric/100`, `!ok→0.0`), meaning it could not validate the redesigned reward. Now it imports and calls `_fix_score_from_completion`, which applies the 3-tier graded scorer. This was the critical prerequisite for a meaningful gate.

2. **Added OFFLINE GATE block** (distinct from the existing HARD GATE) at the end of `main()`. Logic:
   - Compute `n_parseable` = `len(hist_result["parseable_scores"])` (rows that went through rubric scoring)
   - If `n_parseable < 100` → emit `OFFLINE GATE: PROVISIONAL (...)` — blocking, never PASS
   - If `n_parseable >= 100` → evaluate all four threshold conditions and print `OFFLINE GATE: PASS/FAIL` with a before/after table
