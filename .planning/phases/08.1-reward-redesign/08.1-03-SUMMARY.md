## Implementation: Lever 1 Form A

**Problem diagnosed (08.1-MEASUREMENT.md):** 68% parse-failure rate → `else: fix_score = 0.0` assigned all non-parseable completions to the same 0.0 bin as truly empty/no-answer completions. This created an artificial 0-mode responsible for the bimodal distribution and `frac_mid = 0.011` (vs gen-path `frac_mid = 0.202`).

**Fix:** Added `_fix_score_from_completion(completion_text)` helper with three tiers:
1. Empty / no corrected block → `0.0` (preserved low extreme)
2. Non-empty but unparseable → `_PARSE_CLIFF_PARTIAL_CREDIT = 0.25` (de-saturates parse cliff)
3. Parseable PHP → `rubric.overall / 100.0` (preserved high extreme, unchanged)

**Constant chosen:** `0.25` — sits at (0.1, 0.9) frac_mid window center, well below the parseable-high cluster (~0.99 mean), above the 0.1 floor so it contributes to frac_mid. This is a FIXED constant — non-parseable completions do NOT pass through `_extract_verifiable_signals` (that function returns vacuous ~100 on non-code; routing there would reintroduce the reward-hacking surface closed by the original gate, T-81.3-02).

**Preserved semantics:** Security gate path (`_security_fail`) untouched. Gen-path composite untouched (D-81-04). `_is_parseable_php` / `_extract_corrected_php` / `_extract_verifiable_signals` unchanged.

## Implementation: Lever 2 (Weight Rebalance)

Added `judge_consistency_weight_lever2 = 0.45` constant with rationale comment (T-81.3-03 repudiation guard). The call site in `collect_rollouts` now passes `weight=judge_consistency_weight_lever2` to `combine_judge_reward()`.

**Why 0.45:** Amplifies the already-graded consistency signal (frac_mid ~0.73 per Phase 9 handoff §H) as a backstop against the >90 parseable cluster. Within D-09-05 cap (≤ 0.5) — guard NOT relaxed. `combine_judge_reward(weight=0.6)` still raises `ValueError`. Existing test `test_combine_judge_reward_default_weight` continues to pass (it tests the module default 0.3, not the call-site value).

## Implementation: Logging Tiers 4-6 (+ Tier 3 completion)

### Tier 4 — Consistency Score Histogram (`rl_judge_dispatch.py`)
- `_bin_score(s)`: maps [0,1] float to one of 5 bin labels
- `_score_hist`: module-level dict, resets per step
- `get_and_reset_score_hist()`: returns snapshot and resets for next step
- Accumulation wired after imputation pass in `score_judge_consistency_batch`
- Merged into `_log_step` record as `consistency_score_hist`

### Tier 5 — Window Means (`rl_train.py`)
- `compute_window_means(metrics_path, windows)`: verbatim from spec §4 (pandas read_json, NaN-safe)
- Returns keys `window_{lo}_{hi}` with `reward_mean`, `fix_correctness_mean`, `consistency_mean`, `group_reward_std_mean`, `frac_groups_all_zero`, `entropy_mean`
- Called at steps 50/100/150/200/250 via `_WINDOW_TICK_EVERY = 50` in `run_training_step`

### Tier 6 — Kill/Continue Rule (`rl_train.py`)
- `should_flag_for_review(window_means, current)`: verbatim from spec §6
  - `DECISIVE_FLAT`: `w151_200.reward_mean < w0_50.reward_mean - 0.01`
  - `GROUP_COLLAPSE`: `frac_groups_all_zero (w151_200) > 0.5`
  - `ENTROPY_COLLAPSE`: `current["entropy"] < 1.5`
  - `OK`: none of the above
- Logs WARNING when flag fires; failure wrapped in try/except (never halts training)

### Tier 3 — VRAM + Step Wall-Time (`rl_train.py`)
- `vram_used_gb` / `vram_reserved_gb`: `torch.cuda.memory_allocated/reserved()/1e9`
- `vram_fallback_reason`: set when torch unavailable (never dropped)
- `step_wall_time_s`: `perf_counter()` delta from `_step_start` at top of `run_training_step`
- `_log_step` signature extended with `step_start_time=None` (backwards-compatible)
