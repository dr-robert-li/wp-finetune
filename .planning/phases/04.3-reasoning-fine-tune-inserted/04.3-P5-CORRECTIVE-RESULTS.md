`wp-reasoning-v3` ep3 passes **all four** gates and is promoted as the v1.2 reasoning model.

## What changed vs P4

1. **30 `should_fail` training negatives** (`scripts/build_reasoning_negatives.py`) — cot
   format, distinct concrete instances from the 24 held-out sentinel — across the auto-FAIL
   classes (syntax/parse errors, fabricated WP APIs, `$this`/`self`/`parent` out of class,
   SQLi, XSS, missing nonce/cap, recursion, truncation). They teach broken → low relevant
   dims → `overall_score < 40` → `verdict: FAIL`.
2. **Augmented train** = base 563 + 30 = **593** (`scripts/build_augmented_train.py`, leakage
   guard: no negative shares code with sentinel or val). Teacher rows NOT relabeled.
3. **Verdict policy** (`VERDICT-POLICY.md`): PASS iff `overall_score ≥ 70` AND no auto-FAIL
   class; enforced via training + deterministic post-hoc threshold (downstream uses score,
   not the raw verdict token). Fixes the boundary cleanly without prose↔verdict contradiction.
4. **Two-sided gate** (`scripts/check_verdict_confusion.py`) — added because the sentinel is
   one-sided (a FAIL-everything model passes it trivially). Measures recall on teacher-FAIL
   AND false-FAIL on teacher-PASS.

## Two-sided result (the key check)

v3 is a **Pareto improvement**, not a trade: it catches MORE bad code (recall 64%→78%,
sentinel 1→0) AND over-fails LESS good code (false-FAIL 40%→24%). The dreaded
"FAIL-everything" regression did not happen. The one cost is REVL-01A 0.316→0.263 (score-rank
correlation), still well above the 0.171 baseline — an acceptable trade given REVL-05 rejected
on verdict quality, not correlation.
