# Phase 04.3 — Validation Strategy

> Per-phase validation contract for the corrective Tinker MoE-only rank×replay retrain.
> Supersedes the obviated 2026-05-24 local-DGX/Unsloth validation (RTRN-01/02/03 dry-run
> param-walker / checkpoint_parse_check). The live validation surface is: per-candidate
> offline judge gates on the Tinker sampler (cheap pre-merge filter) → merge → REVL-04
> wp-bench HARD gate → confirmation re-run of the winner. See `04.3-RESEARCH.md` §Validation

## Sampling Rate (Nyquist)

- **Per candidate, pre-merge (cheap filter):** offline judge Spearman + REVL-05 sentinel/confusion + FS-terse gate. Only judge-passing candidates advance to the expensive merge+wp-bench.
- **Per candidate, post-merge:** REVL-04 wp-bench (HARD gate ≥ baseline 0.4537).
- **Grid winner:** confirmation re-run (judge + wp-bench) before promotion — D-N5 statistical hardening (wider n / tighter Wilson CI).
- **Format stability (RTRN-05):** FS-terse on cot+ctf ≤ 10% (Wilson-upper ≤ 15%); v3 baseline 0.0%/1.1% must not regress.

## Acceptance Bars (resolved 2026-06-11)

- **Judge accept bar (D-N7):** POINT Spearman ≥ 0.263 (v3 floor / do-no-harm). CI-lower is a
  noise-guard against anomalously-wide-CI measurements only — NOT the accept threshold.
- **Codegen accept bar (D-N8):** REVL-04 wp-bench strictly ≥ baseline 0.4537 (HARD gate). No
  candidate clearing it → escalation decision point, not auto-ship.
- **REVL-05 (non-negotiable):** invalid-PHP sentinel 0 policy false-pass + verdict-confusion gate.
- **Pre-registered selection rule:** max wp-bench codegen subject to judge POINT Spearman ≥ 0.263
  AND REVL-05 gates pass; tie-break on higher judge.
