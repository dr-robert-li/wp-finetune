Pure decision function contract (NEW — added so the selection + escalation logic is unit-testable
without Tinker/merge/wp-bench):
  decide(results: list[dict], wpbench_baseline: float) -> tuple[winner_or_None, exit_code:int]
    - results: list of per-candidate summary dicts (the same shape written to summary.json):
        {"candidate": str, "gates": {judge_spearman:{rho,pass}, sentinel_0_policy:{pass},
         confusion_gate:{pass}, fs_gate:{pass}}, "wpbench_score": float|None, "selected": bool}
    - passing = [c for c in results if c["wpbench_score"] is not None
                 and c["wpbench_score"] >= wpbench_baseline
                 and all four gates pass]    # judge-failing (wpbench_score None) excluded
    - if passing empty -> return (None, 2)    # escalation, no winner
    - else winner = max(passing, key=(wpbench_score, judge rho)); winner["selected"]=True;
           return (winner, 0)
  main() is a THIN wrapper: calls decide(), on exit_code 2 prints the ESCALATION message and
  returns 2; on 0 writes grid_results.json and returns 0. The bars, filter conditions, and
  tie-break are UNCHANGED — only the call site is extracted into an importable function.
