## ⚠️ Vanished-checkpoint discovery (why P4 cost a full re-train)

The prior session reported ep1/2/3 "done" with terse 1.3%/9.1%. Those checkpoints had
**evaporated**: the driver used the deprecated `save_weights_and_get_sampling_client(name=)`,
which produces **ephemeral** sampler weights (`name` is a no-op). A live Tinker query
confirmed **zero persistent checkpoints** (`list_checkpoints`/`list_user_checkpoints` empty;
all runs `n_ckpts=0`). Nothing durable was ever created, so promote/export required re-training.

**Durability fix (committed):** the driver now calls `save_weights_for_sampler(name, ttl_seconds=None)`
(never-expires) + `create_sampling_client(model_path=...)`, plus `save_state()` for the final
checkpoint, and writes the manifest **incrementally every epoch** so a lost process can never
again strand the reference.

## Plan deviation (recorded)

GT-extraction + Spearman + variance-preflight stay byte-identical to the served-vLLM
baseline (numbers are comparable). No merge / no local 30B load / no GB10.
