### dry-run router-freeze walker (NEW helper inside `scripts/train_model.py`) — RTRN-03

**Analog:** `print_training_summary()` (`scripts/train_model.py:401-427`) for the print-block style,
and the dry-run early-return (`scripts/train_model.py:470-472`) for the insertion point.

**The gap (Pitfall 2):** the current dry-run prints a config echo but **NO trainable-parameter count** —
the router freeze is invisible/unverifiable. Current early-return:

```python

    model = apply_lora(model, config)
    ...
    if args.dry_run:
        print("\nDRY RUN MODE — skipping training.")

```

**Pattern to add** (new helper; CALL it in the dry-run branch BEFORE the `return`, after `apply_lora`):

```python
def assert_router_frozen_and_report(model):
    import re
    trainable, total, router_trainable = 0, 0, []
    for name, p in model.named_parameters():
        total += p.numel()
        if p.requires_grad:
            trainable += p.numel()
            if re.search(r"\bmlp\.gate\.", name):          # Qwen3MoeSparseMoeBlock gating linear
                router_trainable.append(name)
    print(f"  Trainable params: {trainable:,} / {total:,} ({100*trainable/total:.4f}%)")
    assert not router_trainable, f"ROUTER NOT FROZEN: {router_trainable[:5]}"
    print("  Router-freeze check PASSED — no mlp.gate.* parameter is trainable (RTRN-03)")
```

**Verification check the planner writes:** dry-run stdout must contain the literal string
`Router-freeze check PASSED` AND a `Trainable params:` line in the single-digit-% range. Run via:
`dgx.execute("unsloth_studio","python","-m","scripts.train_model","--dry-run","--config","config/train_config_reasoning.yaml")`.

> Optional: also echo `warmup_steps` in `print_training_summary` (currently echoes only
> `warmup_ratio` at L420) so the dry-run summary reflects the new field (Validation Map RTRN-01).

### `scripts/checkpoint_parse_check.py` (utility / eval-hook) — NEW (Path C, RTRN-04)

**Reuse import (DO NOT re-implement):** `eval.eval_judge.parse_judge_response`
(`eval/eval_judge.py:112-165`). Keeps abort parse-logic byte-identical to production eval.

**Analog for the abort predicate** — copy the EXACT condition from the production loop
(`eval/eval_judge.py:374-382`):

```python
# eval/eval_judge.py:374-382  [VERIFIED]
parsed = parse_judge_response(generated)
if parsed is None or "overall_score" not in parsed:
    skipped_records.append({"index": i, "reason": "parse_fail", ...})
    continue
```

So `parse_fail` for the new script = `parsed is None or "overall_score" not in parsed`. Use this
exact predicate (not a looser one).

**Why a NEW script and not `eval_judge.py` directly:** `eval/eval_judge.py` is a **vLLM HTTP client**
— its CLI takes `--model` / `--base-url` (`eval/eval_judge.py:602-615`) and calls an OpenAI-compatible
endpoint. It CANNOT score a raw adapter checkpoint in-process. The new script loads the model
in-process instead.

**Analog for in-process model loading:** `scripts/eval_judge_unsloth.py` (the STUB) documents the
intended pattern — `FastLanguageModel.from_pretrained()` binds the MoE expert-LoRA that raw PEFT
cannot — and points at `eval/eval_judge.py`'s `main()` loop as the body to port. Also reuse the
load idiom already proven in `scripts/train_model.py:171-176`:

```python
# load idiom — from scripts/train_model.py:171-176 (merged base) + Unsloth load_adapter for the checkpoint
from unsloth import FastLanguageModel
model, tok = FastLanguageModel.from_pretrained(
    model_name="models/qwen3-30b-wp-30_70-merged",   # merged base (Option B)
    max_seq_length=8192, load_in_4bit=False, dtype=torch.bfloat16,
)
model.load_adapter(checkpoint_dir)                    # bind the in-training reasoning LoRA checkpoint
FastLanguageModel.for_inference(model)
```

**Script skeleton (~80 lines):**
1. `from_pretrained(merged_base)` + `load_adapter(checkpoint_dir)`.
2. Sample 50 `<wp_judge>` prompts from `data/reasoning_dataset/openai_val.jsonl`
   (val file path comes from `config["data"]["val_file"]` — same key as `train_model.py:236`).
3. Generate (temperature 0.0, max_tokens ~1024, matching `eval_judge.py:360-362`).
4. For each, call `parse_judge_response(text)`; count `parse_fail` via the predicate above.
5. Print rate; `sys.exit(1)` (non-zero) if `parse_fail/50 > 0.05` → run-training Step 7 aborts.

**Wiring (NOT a TrainerCallback):** run as a SEPARATE process post-checkpoint in `run-training`
Step 7 (needs the full 57GB model loaded). Run at coarse cadence (every-other `save_steps`) to
bound overhead. CLI args: `--checkpoint-dir`, `--base models/qwen3-30b-wp-30_70-merged`,
`--val-jsonl data/reasoning_dataset/openai_val.jsonl`, `--n 50`, `--threshold 0.05`.
