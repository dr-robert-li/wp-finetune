REQUIRES_ADDITIONAL_ITERATION). 04.3-02 conclusively established that the merged ckpt-72 model
reverts to the BASE model's native judge schema ~37% of the time (terse), that training targets

      format-control weights, OR
  (2) TRAINING-under-imprint — the LoRA adapter itself, applied faithfully, is too weak to
      override the base prior.

This plan runs the discriminator that separates these. The fix differs radically: (1) -> rewrite
`_p0_merge_unsloth_static_moe.py` (no GPU retrain); (2) -> a NEW corrective-training branch
(stronger format signal — NOT the refuted Path-A/Path-B).

The discriminator vehicle is the EXISTING `scripts/checkpoint_parse_check.py` (built in 04.3-01,
RTRN-04): it does in-process Unsloth `FastLanguageModel.from_pretrained(merged-v2)` then
`model.load_adapter(checkpoint-72)`, which binds the fused-MoE expert-LoRA WITHOUT the custom
merge math. Comparing this UNMERGED arm against the MERGED-72 model — both run through the SAME
Unsloth harness, same bf16 precision, same 2048 max-new-tokens, same cot+ctf @120 slice — isolates
the merge math as the only variable.

PRECISION PIVOT (this is the load-bearing correction from the 04.3-03 cross-AI review — see
04.3-REVIEWS.md): the prior 04.3-03 design ran both primary arms at 4-bit on the premise that
"4-bit is the safe default (~16 GiB), bf16 is dangerous (~60 GiB OOM)." Execution on this GB10
unified-memory host proved that premise EMPIRICALLY INVERTED. An on-the-fly 4-bit Unsloth load
double-holds (bf16 shards + a growing 4-bit copy) and transiently PEAKS ~108 GiB — above the
~16 GiB OOM-cascade floor — while a bf16 in-process load is a FLAT ~57 GiB with no double-hold
and is SAFE (validated against primary-source watchdog logs). 4-bit was ALSO scientifically
invalid here: int4 quant of the Qwen3-MoE router was independently established in 04.3-01 to
produce degenerate output (100% parse-fail), which would corrupt the very terse-rate the
discriminator measures. THEREFORE both primary Unsloth arms run at bf16 (`--no-4bit`). This
simultaneously (a) removes the memory blocker, (b) removes the 4-bit-MoE-routing-corruption
confound, and (c) eliminates precision as a cross-arm variable entirely (Arm 1 is already bf16).
The 4-bit pre-quant artifact path (attention-only nf4, transformers-expanded experts, structurally
incompatible with the Unsloth fused-MoE `target_parameters` adapter) is DISCARDED — no 4-bit
checkpoint is on the critical path.
