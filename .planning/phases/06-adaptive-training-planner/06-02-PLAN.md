#### Change 1: Replace MemoryWatchdogCallback with extended version

Replace the entire `MemoryWatchdogCallback` class with a version that:
- Accepts `sample_interval_steps` (default 50) and `thermal_log_path` (default None) in __init__
- Has `_get_sampler()` method that lazy-imports GPUSampler from telemetry.sampler. Returns None if unavailable (prints INFO once, never crashes).
- Has `_sys_ram_mb()` static method returning (total_mb, used_mb, available_mb) from /proc/meminfo (parse MemTotal, MemAvailable, compute used = total - available).
- Has `_write_sample(step, avail_mb)` method that writes ONE JSONL line to self._thermal_log_path with these EXACT field names (canonical schema):
  - `ts`: ISO8601 UTC (datetime.utcnow().isoformat() + "Z")
  - `step`: int (current training step)
  - `gpu_util`: int 0-100 (from GPUSampler, or null)
  - `temp_c`: float (from GPUSampler, or null)  **NOTE: "temp_c" not "temp"**
  - `power_watts`: float or null (from GPUSampler)
  - `vram_used_mb`: int (from GPUSampler, or null)
  - `sys_ram_used_mb`: int (from _sys_ram_mb)
  - `sys_ram_total_mb`: int (from _sys_ram_mb)
  - `mem_available_mb`: int (from _sys_ram_mb)
  - `source`: "watchdog-callback"
  Uses GPUSampler.sample() wrapped in try/except (returns partial record with nulls on failure).
- `on_step_end()`: unchanged 2GB threshold logic for emergency save, PLUS calls _write_sample() when `state.global_step % self.sample_interval_steps == 0` (every 50 steps by default).
- thermal_log_path defaults to env var `ADAPTIVE_THERMAL_LOG` if not passed directly
- OOM_WATCHDOG_THRESHOLD_MB constant stays at 2048

#### Change 2: Add Unsloth override detection via trainer.args (NOT builtins.print)

**CRITICAL: Do NOT monkey-patch builtins.print.** This was removed per reviewer feedback as it is brittle, captures unrelated output, and can break logging.

Instead, AFTER building the trainer via `build_trainer()` but BEFORE calling `trainer.train()`, add this detection block in the `train()` function:

```python
# --- Unsloth override detection (BTCH-02) ---
# Unsloth modifies trainer.args during SFTTrainer.__init__. Read the ACTUAL
# values Unsloth set, compare to our config, and write actuals file if different.
config_batch = config["training"]["per_device_train_batch_size"]
config_accum = config["training"]["gradient_accumulation_steps"]
actual_batch = trainer.args.per_device_train_batch_size
actual_accum = trainer.args.gradient_accumulation_steps

if actual_batch != config_batch or actual_accum != config_accum:
    import json as _json
    print(
        f"\nWARNING: Unsloth overrode training args -- "
        f"batch {config_batch}->{actual_batch}, "
        f"grad_accum {config_accum}->{actual_accum}"
    )
    actuals_path = PROJECT_ROOT / "telemetry" / "training" / "_unsloth_actuals.json"
    actuals_path.parent.mkdir(parents=True, exist_ok=True)
    actuals_path.write_text(_json.dumps({
        "actual_batch": actual_batch,
        "actual_grad_accum": actual_accum,
        "config_batch": config_batch,
        "config_grad_accum": config_accum,
    }))
    print(f"  Actuals written to {actuals_path}")
```

This is more reliable than banner parsing because:
1. It reads the ACTUAL values from the trainer object, not regex-parsed print output
2. It works even if Unsloth changes its banner format
3. It cannot capture unrelated print output
4. No cleanup needed (no monkey-patching to restore)

#### Change 3: Add warmup probe handling with explicit lifecycle

After the `if args.dry_run: return` block in train(), and BEFORE model loading, add:

```python
# --- Warmup probe (PROB-01) ---
probe_flag = PROJECT_ROOT / "telemetry" / "training" / "_warmup_probe_required"
if probe_flag.exists():
    import json as _json
    probe_data = _json.loads(probe_flag.read_text())
    proposed_batch = probe_data.get("proposed_batch")

    # Read probe steps from adaptive config
    try:
        adaptive_cfg = yaml.safe_load(
            open(PROJECT_ROOT / "config" / "adaptive_planning.yaml")
        )
        probe_steps = adaptive_cfg.get("probe", {}).get("steps", 5)
    except Exception:
        probe_steps = 5

    print(f"\n{'='*60}")
    print(f"  WARMUP PROBE: Testing batch_size={proposed_batch} for {probe_steps} steps")
    print(f"{'='*60}\n")

    try:
        from telemetry.probe import prepare_probe, evaluate_probe

        # Prepare and run probe (loads model, runs N steps, collects telemetry)
        probe_ctx = prepare_probe(config, proposed_batch, probe_steps)

        # Get thermal log path for probe output
        thermal_log = os.environ.get("ADAPTIVE_THERMAL_LOG",
            str(PROJECT_ROOT / "telemetry" / "training" / "thermal_probe.jsonl"))

        result = evaluate_probe(probe_ctx, thermal_log)

        # EXPLICIT GPU CLEANUP between probe and full run
        # Model is NOT reused -- full reload for real training.
        if hasattr(probe_ctx, 'model'):
            del probe_ctx.model
        if hasattr(probe_ctx, 'trainer'):
            del probe_ctx.trainer
        del probe_ctx
        import gc
        gc.collect()
        if torch.cuda.is_available():
            torch.cuda.empty_cache()
            torch.cuda.synchronize()

        if result.ok:
            print(f"  PROBE PASSED: batch_size={proposed_batch} is safe")
            probe_flag.unlink()  # Consumed -- delete sentinel
        else:
            print(f"  PROBE FAILED: {result.reason}")
            probe_flag.unlink()  # Consumed
            # Write failure artifact for adaptive_planner to consume next cycle
            fail_path = PROJECT_ROOT / "telemetry" / "training" / "_probe_failed"
            fail_path.write_text(_json.dumps({
                "proposed_batch": proposed_batch,
                "reason": result.reason,
            }))
            print(f"  Failure recorded at {fail_path}")
            print("  Reverting to previous config and continuing with normal training.")
            # Do NOT sys.exit -- let training proceed with current (pre-change) config

    except ImportError:
        print("  WARNING: telemetry.probe not available (dgx-toolbox Phase 13 not mounted)")
        print("  Falling back to 1-step MemAvailable check (degraded mode)")
        # Degraded probe: just check if MemAvailable > threshold after loading
        probe_flag.unlink()  # Consumed even in degraded mode
```

Place this block AFTER dry_run check but BEFORE the main model loading sequence (check_memory, load_model_and_tokenizer, etc.). The probe loads its own model independently; after probe completes (pass or fail), GPU memory is explicitly cleaned, and the main training flow proceeds with a fresh model load.

**Also update build_trainer()**: Read sample_interval_steps and callback_threshold_mb from config/adaptive_planning.yaml (with try/except fallback to defaults). Pass to MemoryWatchdogCallback constructor. Read ADAPTIVE_THERMAL_LOG from os.environ and pass as thermal_log_path.
