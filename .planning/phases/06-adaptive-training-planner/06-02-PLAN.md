Make FOUR targeted edits to `scripts/train_model.py`. Read the full file first to understand existing structure, then apply changes in this order:

#### Change 1: Add page cache drop to check_memory()

At the top of `check_memory()`, before the /proc/meminfo read, add:
```python
import os as _os
_os.system("sudo sync && echo 3 | sudo tee /proc/sys/vm/drop_caches > /dev/null 2>&1")
```
This eliminates page cache jitter from baseline measurements. Add a comment referencing NVIDIA forum confirmation that CUDA tools can misread cached memory as occupied on UMA.

#### Change 2: Replace MemoryWatchdogCallback with extended version

Replace the entire `MemoryWatchdogCallback` class with a version that:
- Accepts `sample_interval_steps` (default 50) and `thermal_log_path` (default None) in __init__
- Has `_get_sampler()` method that lazy-imports GPUSampler from telemetry.sampler (prints INFO if unavailable, never crashes)
- Has `_sys_ram_mb()` static method returning (total_mb, used_mb, available_mb) from /proc/meminfo
- Has `_write_sample(step, avail_mb)` that writes one JSONL line to self._thermal_log with fields: ts (ISO8601 UTC), step, gpu_util, temp, power_watts, vram_used_mb, sys_ram_used_mb, sys_ram_total_mb, mem_available_mb, source="watchdog-callback". Uses GPUSampler.sample() to get gpu_util, temp_c, power_watts -- wraps in try/except.
- `on_step_end()`: unchanged 2GB threshold logic, PLUS calls _write_sample() every sample_interval_steps (when state.global_step % self.sample_interval_steps == 0)
- thermal_log_path defaults to env var ADAPTIVE_THERMAL_LOG if not passed directly
- OOM_WATCHDOG_THRESHOLD_MB constant stays at 2048

#### Change 3: Add Unsloth banner parsing around trainer.train()

In the `train()` function, wrap the `trainer.train(resume_from_checkpoint=resume_checkpoint)` call with:
- Define `_banner_pattern = re.compile(r"Batch size per device\s*=\s*(\d+).*Gradient accumulation steps\s*=\s*(\d+)", re.IGNORECASE)`
- Monkey-patch builtins.print to intercept Unsloth's banner output
- When pattern matches, write actuals to telemetry/training/_unsloth_actuals.json with keys: actual_batch, actual_grad_accum, config_batch, config_grad_accum
- Log WARNING if actual != config
- Restore builtins.print in a finally block

#### Change 4: Add warmup probe handling after dry_run check

After the `if args.dry_run: return` block in train(), add:
- Check for PROJECT_ROOT / "telemetry/training/_warmup_probe_required"
- If exists, read probe_data JSON, get proposed_batch
- Read PROBE_STEPS from config/adaptive_planning.yaml (probe.steps, default 5)
- Try importing from telemetry.probe: prepare_probe, evaluate_probe
- Run prepare_probe(), build a mini-trainer with max_steps=PROBE_STEPS, run probe, evaluate_probe()
- If probe passes: delete flag, proceed
- If probe fails: delete flag, write _probe_failed JSON with proposed_batch and reason
- If telemetry.probe import fails: fallback to 1-step MemAvailable check (degraded mode)

**Also update build_trainer()**: Read sample_interval_steps and callback_threshold_mb from config/adaptive_planning.yaml (with try/except fallback). Pass to MemoryWatchdogCallback constructor. Read ADAPTIVE_THERMAL_LOG from os.environ.
