**Header:**
```markdown
# Skill: wp-finetune:adaptive-planner

Adaptive resource planning decision engine (v4.0). Invoked by run-training between runs to update
train_config.yaml for the next run. Implements power-primary routing, thermal exploitation ladder,
batch/grad_accum coupling, and worker scaling.

**This skill is NOT invoked by the user directly.** run-training calls it at Step 8.5.

**Prerequisite:** dgx-toolbox Phase 13 (telemetry/ package) must be complete and mounted.

## Trigger

Called by run-training skill at Step 8.5 with these variables in scope:
- `$THERMAL_LOG` -- path to canonical JSONL for the completed run
- `$RATIO` -- dataset ratio just completed (e.g., "30_70")
- `$TELEMETRY` -- true/false (if false, skip entirely)
```

**Step 1: Guard clause** -- If $TELEMETRY is false, print message and exit. Check that dgx-toolbox telemetry modules are importable (GPUSampler, compute_effective_scale, AnchorStore, classify_failure). If ImportError, print FATAL message with troubleshooting hint and sys.exit(1).

**Step 2: Load config and thresholds** -- Load training config from config/train_config.yaml (or ratio-specific overlay). Load adaptive planning thresholds from config/adaptive_planning.yaml. Set convenience aliases: pz = power_zones, tb = thermal_brake, mem_cfg = memory, probe_cfg = probe, ladder_cfg = ladder, worker_cfg = worker_budget. Extract current: batch_size, grad_accum, effective_batch (= batch_size * grad_accum), workers, prefetch, persistent_workers, save_steps, eval_steps.

**Step 3: Read canonical telemetry JSONL** -- Parse $THERMAL_LOG line by line. Extract gpu_utils, gpu_temps, power_watts, sys_ram_used, sys_ram_total, mem_available arrays. Compute aggregates: avg_gpu_util, peak_gpu_util, avg_temp, peak_temp, avg_watts, peak_watts. Compute memory: mem_total_gb, peak_ram_gb, p95_ram_gb, safe_headroom_gb. Apply jitter_margin (memory.jitter_margin_gb = 5) to get effective_headroom_gb. Track min_mem_available_gb.

**Step 4: Check for Unsloth banner override** -- Read telemetry/training/_unsloth_actuals.json if it exists. If Unsloth overrode batch or grad_accum (actual != config), log WARNING, update batch_size/grad_accum/effective_batch to actuals, set unsloth_override=True. Delete actuals file after consuming. Per BTCH-02, BTCH-03.

**Step 5: Failure classification** -- Call classify_failure(readings, final_n=5, oom_gpu_util_threshold=10, oom_ram_pct_threshold=95, hang_gpu_util_threshold=5, hang_cpu_busy_threshold=80). Returns "NORMAL"|"OOM"|"HANG"|"THERMAL". OOM jumps to OOM recovery (Step 8b). HANG logs and exits (no config change, records hard cap in anchor store). THERMAL handled as THROTTLED in Step 6.

**Step 6: Power-zone routing (primary dispatch per ADPT-01)** -- This is the core of v4.0. Route by avg_watts:
- If avg_watts is None (no power data), fall back to GPU util as proxy: <30% = UNDERUTILIZED, <70% = TARGET, <90% = MODERATE, else CAPPED
- If avg_watts available: >= capped_min_watts (80W) = CAPPED, >= target_min_watts (50W) = TARGET, < 50W = UNDERUTILIZED
- TARGET refined to MODERATE if has_batch_headroom AND effective_headroom_gb > 0
- THERMAL OVERRIDE: if peak_temp >= thermal_brake.warning_temp (82C), force power_zone = "THROTTLED" regardless of watts. If peak_temp >= thermal_brake.emergency_temp (85C), print EMERGENCY.
- Route: THROTTLED -> force batch to 1, CAPPED -> hold config, TARGET -> hold or minor, MODERATE -> cautious climb, UNDERUTILIZED -> aggressive climb

**Step 7: Effective scale computation** -- Call compute_effective_scale(raw_params_b=30.0, quant_mode="bf16_lora", checkpoint_type="lora", seq_len=4096, lora_rank=32, optimizer="adamw", moe_active_ratio=0.10). Extract batch_ceiling, min_headroom_pct, min_headroom_gb. Determine step_size: ladder.step_size_small (2) if <=13B effective, else ladder.step_size_large (1).

**Step 8: Apply routing decision (per ADPT-02 ladder order)**:
- 8a THROTTLED: new_batch=1, new_accum=effective_batch, touch _thermal_cooldown_required with cooldown_target
- 8b OOM recovery: restore from AnchorStore.last_safe_config() or conservative defaults (batch=2, workers=2), set persistent_workers=True (sticky)
- 8c Thermal exploitation ladder (MODERATE/UNDERUTILIZED zones):
  - I/O Bottleneck Guard: if gpu_util <30% AND watts <30W, pause Rung 1
  - RUNG 1 (batch): if not io_bottleneck, below ceiling, headroom OK, not in cooldown, zone is MODERATE/UNDERUTILIZED -> propose batch + step_size. If anchor exists -> apply. If no anchor -> apply AND write _warmup_probe_required flag (per PROB-01)
  - RUNG 2 (prefetch): increment by 1 up to ladder.prefetch_cap (4)
  - RUNG 3 (workers): budget formula workers * prefetch * avg_sample_mb < worker_budget.max_gb * 1024. Hard cap: worker_budget.hard_cap_uma (6). Decrease if MemAvailable <5GB for 2 consecutive runs. Increase if util <70%, budget allows, hard cap OK, headroom >= min_headroom + 5%
  - RUNG 4 (save_steps): double up to ladder.save_steps_cap (400)
  - RUNG 5 (eval_steps): double up to ladder.eval_steps_cap (200)
- 8d Sticky settings: persistent_workers once enabled never auto-disabled. prefetch only increases. pin_memory always False on Spark (UMA). Hard caps from anchor store never auto-removed.
- Per BTCH-01: Every batch change uses new_accum = max(1, effective_batch // new_batch)

**Step 9: Write updated config, anchor, and log** -- Write changes to config/train_config.yaml via yaml.dump. Record in AnchorStore (per PROB-02). Append structured log entry to telemetry/training/adaptive_adjustments.md with: power_zone, failure_mode, gpu_util, memory stats, effective_scale, unsloth_override, all config changes with old->new values, reason.

**Troubleshooting section** at the end covering: "dgx-toolbox telemetry/ not importable", "No power_watts in telemetry", "effective_batch changed -- WARNING", "HANG detected -- no config change".
