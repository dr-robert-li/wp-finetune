Create `scripts/adaptive_planner.py` as a pure Python module. This addresses the HIGH review concern about untestable skill logic.

The module MUST contain these functions:

1. `classify_power_zone(avg_watts, peak_temp, avg_gpu_util, thresholds, has_batch_headroom=False, has_mem_headroom=False) -> str`
   - Returns one of: "THROTTLED", "CAPPED", "TARGET", "MODERATE", "UNDERUTILIZED"
   - Thermal brake: if peak_temp >= thresholds["thermal_brake"]["warning_temp"] (82) -> "THROTTLED" regardless of watts
   - If avg_watts is None, fall back to gpu_util proxy: <30% -> UNDERUTILIZED, <70% -> TARGET, <90% -> MODERATE, else CAPPED
   - Normal routing: avg_watts >= capped_min_watts (80) -> CAPPED; >= target_min_watts (50) -> TARGET or MODERATE (if headroom); < 50 -> UNDERUTILIZED
   - TARGET refines to MODERATE when has_batch_headroom AND has_mem_headroom are both True

2. `couple_batch_grad_accum(effective_batch, new_batch, max_drift=1) -> int`
   - Addresses HIGH review concern: use `round()` not `//`. Constrain to divisors when possible.
   - First, check if effective_batch is exactly divisible by new_batch. If yes, return effective_batch // new_batch.
   - If not, use `round(effective_batch / new_batch)`.
   - Result must be >= 1.
   - Verify abs(new_batch * result - effective_batch) <= max_drift. If drift exceeds max_drift, log warning but still return.

3. `compute_batch_ceiling(model_config, lora_config) -> dict`
   - Calls `from telemetry.effective_scale import compute` (NOT compute_effective_scale -- that function does not exist)
   - For Qwen3-30B-A3B: raw_params=30e9, quant_mode="bf16", gradient_checkpointing_mode="full" (Unsloth "unsloth" mode ~ full), lora_rank from config, seq_len from config, optimizer="adamw"
   - Returns dict with keys: effective_params, batch_cap, min_headroom_pct

4. `apply_ladder(current_config, power_zone, thresholds, telemetry_summary, anchor_lookup_fn=None) -> dict`
   - Takes current training config dict, power zone string, thresholds dict, telemetry summary dict
   - Returns dict with new config values and reason strings
   - Evaluates rungs in v4.0 order: batch (1) > prefetch (2) > workers (3) > save_steps (4) > eval_steps (5)
   - I/O Bottleneck Guard: if avg_gpu_util < thresholds io_util_threshold AND avg_watts < io_watts_threshold -> skip Rung 1
   - Rung 1 (batch): increase by step_size (1 for >13B, 2 for <=13B) up to batch_ceiling. Calls couple_batch_grad_accum.
   - Rung 2 (prefetch): +1 up to prefetch_cap (4)
   - Rung 3 (workers): +1 if budget allows (workers * prefetch * avg_sample_mb < max_gb * 1024) and hard_cap_uma not reached
   - Rung 4 (save_steps): *2 up to save_steps_cap (400)
   - Rung 5 (eval_steps): *2 up to eval_steps_cap (200)
   - Sticky settings: persistent_workers once true never reverts; prefetch never decreases; pin_memory always False on UMA

5. `parse_telemetry_jsonl(jsonl_path) -> dict`
   - Reads canonical JSONL, returns summary dict with: avg_watts, peak_watts, avg_gpu_util, peak_temp, avg_temp, mem_total_gb, peak_ram_gb, effective_headroom_gb, min_mem_available_gb
   - Skips first 2 readings (warm-up noise per Gemini suggestion)
   - Returns None if fewer than 3 valid readings after skip
