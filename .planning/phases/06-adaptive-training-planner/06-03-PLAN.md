### Step 5.5: Clear page cache (optional, recommended)

**This is a manual pre-run action, not automated.** On UMA systems like DGX Spark,
Linux page cache can consume 5+ GB, causing memory pre-checks to underestimate available RAM.

If running on the host (not inside container):
```bash
sudo sync && echo 3 | sudo tee /proc/sys/vm/drop_caches > /dev/null 2>&1
```

If `sudo` is not available or the environment is containerized, skip this step.
The adaptive planner accounts for page cache jitter via the `memory.jitter_margin_gb`
threshold (5 GB) in config/adaptive_planning.yaml.
```
