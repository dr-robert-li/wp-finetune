  <action>
**DO NOT run `python -m scripts.phase2_generate` — it calls the Anthropic API.**

Instead, spawn Claude Code agents that generate synthetic WordPress code to fill taxonomy gaps.

**Step 1: Read the gap report** to understand what needs generating. Parse phase2_synthetic/gap_report.json to identify gap tags and deficit counts.

**Step 2: Read config/synthetic_prompts.yaml** for prompt templates per gap tag, including:
- Standard templates (templates section) — keyed by gap tag
- Contrastive templates (contrastive_templates section) — bad->good pairs
- Rejection templates (rejection_templates section) — proactive_nonce, proactive_capability, proactive_escaping (~500 total)

**Step 3: Read 3-5 passed functions from phase1_extraction/output/passed/** as style anchors. These show the quality bar and coding style the synthetic examples must match.

**Step 4: Spawn 4 parallel Claude Code agents**, each assigned a subset of gap tags to fill:

Each agent:
1. Receives: a list of gap tags with deficit counts, the synthetic_prompts.yaml templates, and 2-3 style anchor examples
2. For each gap tag, generates the deficit count number of synthetic examples using the templates with varied {complexity}, {context}, {constraint} parameters
3. Each generated example is a JSON object:
```json
{
  "function_name": "descriptive_name",
  "source_repo": "synthetic",
  "source_file": "synthetic/{gap_tag}.php",
  "body": "<?php\n... the generated code ...",
  "quality_tier": "synthetic",
  "training_tags": ["gap_tag", ...],
  "generation_template": "template text used",
  "complexity": "simple|intermediate|advanced|production-scale"
}
```
4. Writes output to phase2_synthetic/output/generated/{gap_tag}.json — one file per gap tag

**Agent assignment:** Divide gap tags roughly equally by total deficit count across 4 agents. One agent should specifically handle all rejection_templates (target ~500 rejection examples total, split roughly: 170 proactive_nonce, 170 proactive_capability, 160 proactive_escaping).

**For rejection examples specifically:**
- The prompt asks for code WITHOUT mentioning security
- The generated response MUST proactively add nonces/capability checks/escaping
- Tag with "rejection:proactive_nonce", "rejection:proactive_capability", or "rejection:proactive_escaping"
- These are wp_gen examples (the model learns to add security unprompted)

**Quality requirements for all generated code:**
- Must follow WordPress Coding Standards (WPCS)
- Must use $wpdb->prepare() for all dynamic SQL
- Must escape output, verify nonces, check capabilities
- Must include PHPDoc with @param, @return, @since
- Must handle edge cases
- Must be realistic, production-quality WordPress PHP
