**Spawn 5 parallel Claude Code agents.** Each agent:

1. Receives a batch of repo names to process
2. For each repo, reads phase1_extraction/output/extracted/{repo}.json
3. For each function in the JSON array, evaluates it against the judge rubric in config/judge_system.md
4. Produces an assessment object with EXACTLY this structure:
```json
{
  "function_name": "...",
  "file_path": "...",
  "verdict": "PASS" or "FAIL",
  "scores": {
    "wpcs_compliance": 1-10,
    "sql_safety": 1-10,
    "security": 1-10,
    "performance": 1-10,
    "wp_api_usage": 1-10,
    "code_quality": 1-10,
    "dependency_integrity": 1-10,
    "i18n": 1-10,
    "accessibility": 1-10
  },
  "critical_failures": [],
  "dependency_chain": [],
  "training_tags": [],
  "notes": "..."
}
```
5. PASS criteria: ALL scores >= 8 AND no critical failures. Security auto-FAIL: security < 5 forces FAIL.
6. N/A scoring: i18n and accessibility score 7 (not 10) when function has no relevant output.
7. Adds the assessment to each function object as an `assessment` field
8. Also copies training_tags from assessment to the top-level function object
9. Splits functions into passed array (verdict=PASS) and failed array (verdict=FAIL)
10. Writes phase1_extraction/output/passed/{repo}.json (passed functions) and phase1_extraction/output/failed/{repo}.json (failed functions)
