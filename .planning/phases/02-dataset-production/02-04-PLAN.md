  <action>
Run the Phase 1 judge script from the project root:

```bash
cd /home/robert_li/Desktop/projects/wp-finetune && python -m scripts.phase1_judge
```

This script:
1. Reads extracted functions from phase1_extraction/output/extracted/
2. Runs PHPCS pre-filter (< 5 errors/100 lines threshold)
3. Sends passing functions to Claude API for rubric-based judging (threshold >= 8 per user decision)
4. Applies security auto-FAIL (any security dimension < 5 = FAIL)
5. Routes batches of >= 50 functions through Batch API (50% cost savings)
6. Writes passed functions to phase1_extraction/output/passed/ and failed to output/failed/
7. Uses checkpoint/resume so it can survive interruptions

CRITICAL: This is a long-running API-heavy script. It will make many Claude API calls (or Batch API submissions) to judge functions across 56 repos. Expect this to take 1-4+ hours depending on function count and Batch API processing time.

The script loads ANTHROPIC_API_KEY from .env via python-dotenv. No manual env setup needed.

If the script is interrupted, re-run the same command -- checkpointing skips already-judged repos.

Monitor the output for:
- "BATCH" messages indicating Batch API usage
- Pass/fail counts per repo
- Any error messages about API failures (backoff will retry automatically)
