**Seed loading and sampling:**
- `load_seeds()` — loads both `ugc_seeds.json` and `ugc_boundary_seeds.json`, filters to `seed_type == "deep_judge_cot"`, returns list
- `sample_seeds(seeds, n=3)` — randomly samples n seeds; weights boundary seeds 2x higher (use `random.choices` with weights based on `defect_subtlety == "boundary"`)

**Few-shot exemplar formatting:**
- `format_seed_as_exemplar(seed)` — formats a deep_judge_cot seed as text showing code snippet (truncated to 1500 chars), then reasoning: verdict, dimension_analysis (all entries), overall_score, key_observation. Format matches research Pattern 1.

**Source function loading:**
- `load_phase1_functions()` — scans PASSED_DIR and FAILED_DIR, loads each JSON file, extracts individual functions. Each function becomes a dict with `code`, `source_file`, `source_dir` ("passed" or "failed"), `function_name`. Filters out functions with code length < 50 chars (per research pitfall 6 — skip empty/minimal functions). Returns list of all functions.

**Core generation function:**
- `generate_deep_judge_cot(code, source_info, seeds, client)` — takes a PHP code string, source metadata, list of CoT seeds, and anthropic client. Samples 3 seeds via `sample_seeds()`. Constructs prompt with:
  1. System instruction: "You are a WordPress code quality assessor producing deep reasoning chains for training data. You MUST analyze ALL 9 dimensions regardless of how many the example seeds show. Seeds only show 2-3 dimensions as examples of analysis depth, not as the complete set."
  2. The 3 formatted seed exemplars
  3. "NOW ANALYZE:" block with the target code (truncated to 3000 chars)
  4. Explicit output schema instruction: "Return JSON with keys: verdict (PASS/FAIL), dimension_analysis (object with ALL 9 dimensions: wpcs_compliance, sql_safety, security, performance, wp_api_usage, code_quality, dependency_integrity, i18n, accessibility — each having score (1-10) and analysis (string citing specific WordPress APIs by name)), overall_score (0-100), key_observation (string)."
  5. Explicit API citation instruction: "When WordPress APIs appear in the code or are missing from the code, name them explicitly: $wpdb->prepare(), wp_verify_nonce(), esc_html(), current_user_can(), check_ajax_referer(), esc_attr(), esc_url(). Do not describe behavior without naming the API."

- Calls `call_with_backoff(client, model="claude-sonnet-4-6-20250514", max_tokens=2048, messages=[{"role": "user", "content": prompt}])`
- Extracts result via `extract_json(resp.content[0].text)`
- Returns the parsed dict or None

**Quality gate per example:**
- `passes_quality_gate(result)` — checks:
  1. `verdict` is "PASS" or "FAIL"
  2. `dimension_analysis` has all 9 required dimensions
  3. Each dimension has `score` (int 1-10) and `analysis` (non-empty string)
  4. `overall_score` is int 0-100
  5. Returns True/False

**Training example formatter:**
- `format_training_example(source_info, result)` — assembles the final training example dict with fields: `source_file`, `source_dir`, `function_name`, `code`, `reasoning` (the full result dict), `dimensions_addressed` (always all 9), `generation_method` ("seed_few_shot_agent")

**Main function with argparse:**
- `--pilot` flag: generates 40 examples (writes to PILOT_DIR / "deep_judge_cot_pilot.json")
- `--target N` flag: override bulk target count
- Default bulk target: `int(len(all_functions) * 0.5)` — aim for ~200 examples
- Uses `load_checkpoint("generate_deep_judge_cot")` and `save_checkpoint` every 20 examples
- Tracks `parse_attempts` and `parse_failures`; prints warning if failure rate > 2%
- At end, writes output JSON array to the appropriate directory
- Prints summary: total generated, parse failure rate, dimension coverage stats

**Pilot validation (when --pilot):**
- After generating pilot batch, run `validate_pilot_batch(examples)` that checks:
  1. All 9 dimensions appear across examples
  2. At least 3 distinct WP_API_CITATIONS appear in reasoning text across all examples
  3. Print results: missing dimensions, API citations found, per-dimension score distribution
- Exit with code 0 on success, code 1 if missing dimensions

**Output directory creation:**
- Create `data/phase4_reasoning/`, `data/phase4_reasoning/pilot/`, `data/phase4_reasoning/deep_judge_cot/` directories with `exist_ok=True`
