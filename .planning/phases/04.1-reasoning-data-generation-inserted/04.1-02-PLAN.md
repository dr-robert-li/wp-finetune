**Seed loading:**
- `load_critique_seeds()` — loads both seed files, filters to `seed_type == "critique_then_fix"`, returns list (expected 59 seeds)
- `sample_seeds(seeds, n=3)` — same boundary-weighted random sampling as deep_judge_cot.py

**Few-shot exemplar formatting for critique seeds:**
- `format_critique_seed_as_exemplar(seed)` — formats showing: defective_code (truncated 1500 chars), then human_critique with summary, dimensions dict (severity + issue + fix per dimension), key_observation, then corrected_code in `<corrected_code>` tags

**Source function loading:**
- `load_failed_functions()` — scans FAILED_DIR only (per D-02, critique-then-fix uses failed functions, NOT mutation pool). Loads each JSON file, extracts individual functions. Each becomes dict with `code`, `source_file`, `function_name`. Filters out functions with code length < 50 chars. Returns list.

**Core generation function:**
- `generate_critique_then_fix(code, source_info, seeds, client)` — takes defective PHP code, source metadata, list of critique seeds, and anthropic client. Samples 3 seeds. Constructs prompt with:
  1. System instruction: "You are a WordPress code quality critic producing structured critiques for training data. You receive defective PHP code and must: (a) produce a structured critique analyzing ALL 9 dimensions with severity (critical/high/medium/low), issue description, and fix suggestion for each; (b) produce the corrected version of the code wrapped in <corrected_code> XML tags."
  2. The 3 formatted critique seed exemplars
  3. "NOW CRITIQUE AND FIX:" block with the target defective code (truncated to 3000 chars)
  4. Explicit output schema: "Return JSON with keys: summary (string), dimensions (object with ALL 9 dimensions: wpcs_compliance, sql_safety, security, performance, wp_api_usage, code_quality, dependency_integrity, i18n, accessibility — each having severity (critical|high|medium|low), issue (string), fix (string)), key_observation (string), corrected_code (string containing the fixed PHP code). IMPORTANT: The corrected_code value must be the actual corrected PHP source, not wrapped in additional tags."
  5. WP API citation instruction: "When WordPress APIs appear in the defective code or are needed in the fix, name them explicitly: $wpdb->prepare(), wp_verify_nonce(), esc_html(), current_user_can(), etc."

- Calls `call_with_backoff(client, model="claude-sonnet-4-6-20250514", max_tokens=3072, messages=[{"role": "user", "content": prompt}])` (higher max_tokens than CoT because corrected code is included)
- Extracts result via `extract_json(resp.content[0].text)`
- If `corrected_code` not in extracted JSON, attempts fallback extraction: search for `<corrected_code>` XML tags in the raw response text via regex `re.search(r'<corrected_code>([\s\S]+?)</corrected_code>', resp.content[0].text)` and inject into result dict
- Returns the parsed dict or None

**Quality gate per example:**
- `passes_quality_gate(result)` — checks:
  1. `summary` is non-empty string
  2. `dimensions` has all 9 required dimensions
  3. Each dimension has `severity` in SEVERITY_LEVELS, `issue` (non-empty), `fix` (non-empty)
  4. `corrected_code` is non-empty string with length > 20
  5. Returns True/False

**Training example formatter:**
- `format_training_example(source_info, result)` — assembles: `source_file`, `function_name`, `defective_code` (from source), `critique` (summary + dimensions + key_observation from result), `corrected_code` (from result), `dimensions_addressed` (always all 9), `generation_method` ("seed_few_shot_agent")

**Pilot validation:**
- `validate_pilot_batch(examples)` — checks:
  1. All 9 dimensions appear across examples (check critique.dimensions keys)
  2. All 4 severity levels appear at least once across examples
  3. At least 3 WP_API_CITATIONS appear in critique text
  4. corrected_code is non-empty for all examples
  5. Returns dict with missing_dimensions, severity_coverage, api_citations_found

**Main function with argparse (mirrors deep_judge_cot.py):**
- `--pilot`: generates 40 examples, writes to PILOT_DIR / "critique_then_fix_pilot.json"
- `--target N`: override bulk target
- Default bulk target: `int(len(failed_functions) * 0.5)`
- Checkpoint/resume via `load_checkpoint("generate_critique_then_fix")` and `save_checkpoint` every 20 examples
- Tracks parse_attempts and parse_failures
- Prints summary at end
