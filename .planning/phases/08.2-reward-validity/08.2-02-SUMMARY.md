## What Was Built

### scripts/rl_codegen_tripwire.py

- `CODEGEN_BAR_V12`: sourced from `_rlev01_wpbench_ckpt.BASELINE_V12` (single source of truth; T-082-05 CI guard in test asserts 0.4616).
- `KNOWLEDGE_FLOOR = 0.45`, `EXEC_FLOOR = 0.375`: from Phase-10 regression report (step-500 breach).
- `check_codegen_tripwire(wpbench_score, bar, sub_scores)`: pure decision function, injectable, no GPU. Returns halt-reason string below bar or on sub-floor breach; None when score is None or at/above bar.
- `run_codegen_probe(model_dir, tag, step)`: lazy-imports `_wpbench_with_boot` from `scripts.run_eval_reasoning`. Never called in any offline test or dry-run. Exercised only by Plan 05's live smoke.

### scripts/rl_train.py

- Imports `CODEGEN_BAR_V12`, `check_codegen_tripwire`, `run_codegen_probe` at module level (import-safe).
- `run_training_step`: after `check_halt`, adds codegen probe block gated by `getattr(args, "codegen_probe_every", 0) > 0 and step % probe_every == 0`. Folds result via `halt_reason = halt_reason or codegen_halt` — single halt mechanism (CR-04 ordering preserved; optim_step never called on halt).
- `_parse_args`: adds `--codegen-probe-every` (int, default 0), `--codegen-bar` (float, default CODEGEN_BAR_V12), `--codegen-score-override` (float, default None; test/dry seam only).

### tests/test_rl_codegen_tripwire.py

- `test_decision_only`: below-bar → reason; above-bar → None; at-bar → None; None-score → None; sub-floor breaches (exec, knowledge) → reason even if overall passes.
- `test_args_wired`: all three new flags parse with correct defaults.
- `test_tripwire_fires_below_bar`: `run_training_step` with `codegen_score_override=0.40` → `halted=True`, emergency checkpoint saved, `optim_step` NOT called.
- `test_tripwire_silent_above_bar`: `codegen_score_override=0.50` → `halted=False`, `optim_step` called.

## Known Stubs / Deferred Live-Path Wiring

`run_codegen_probe` is wired and importable, but two live-path inputs are placeholder / unverified offline and deferred to Plan 05:

1. **checkpoint dir** — `model_dir` is obtained via `getattr(args, "codegen_probe_model_dir", ".")`. The sentinel `"."` is a placeholder; nothing in `_parse_args` wires a real checkpoint path yet. Plan 05 must add `--codegen-probe-model-dir` or materialise a merged checkpoint dir before calling the live path.

2. **sub-score key names** — `_codegen_sub` extracts `"knowledge"` and `"exec"` from the `_wpbench_with_boot` result dict. The actual key names in `run_eval_reasoning._wpbench_with_boot`'s return value are unverified offline. If the real keys differ (e.g. `knowledge_score`), the sub-floors silently become `None` and never fire. Plan 05's smoke must confirm key names and update the extraction if needed.

**Checkpoint-timing design note for Plan 05:** The probe block runs *before* `optim_step` (correct per CR-04 — halt before committing the divergent gradient). At probe time there is deliberately **no** just-saved checkpoint from the current step. Plan 05 must materialise a checkpoint of the pre-update weights at probe time (e.g. via `save_weights_and_get_sampling_client` or a dedicated pre-probe save) before calling `run_codegen_probe`. The `model_dir="."` placeholder stands in for this missing materialisation step. This is a deliberate design choice, not a bug.
