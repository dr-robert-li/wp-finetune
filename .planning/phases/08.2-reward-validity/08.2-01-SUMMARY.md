# Phase 08.2 Plan 01: Reward Validity Gate Formalization Summary

**One-liner:** Formalized the oracle into a reusable standing gate (`run_validity_gate` -> `GateResult`), joined 342 TRAIN-GT rows to the RL judge pool by content-hash, and regression-locked the known oracle verdicts (fix_correctness INVALID, pairwise_rank_agreement VALID) in 4 pytest tests.

## What Was Built

### Task 1: TRAIN Teacher-GT Sidecar (Keystone Gate)

`scripts/build_reward_gt_sidecar.py` — CPU/$0, no API. Recovers teacher GT from
`openai_train.jsonl` (TRAIN split only) and joins to the RL judge pool by
**content-hash** of whitespace-normalized PHP code-under-review (SHA-256[:16]).

Coverage achieved:
- 482 wp_judge rows in openai_train.jsonl — 342 GT-extractable (140 had no `overall` field)
- 342/482 RL pool prompt_ids matched via hash (no PHP-extraction failures)
- 93/93 probe corpus prompt_ids covered by the sidecar
- Blocking gate: 342 >= 60 distinct prompt_ids — **PASSED**

Output: `data/rl_probe/judge_gt_sidecar.jsonl` — 342 rows `{prompt_id, code_hash, teacher_overall, source:"train"}`.

Anti-leakage: `source=="train"` on every row (val is the oracle's held-out target; using it would make SC2 circular and violate T-09-LEAK).

### Task 2: Importable Oracle + Gate Entrypoint + Gate Rule

`scripts/_reward_validity_oracle.py` — minimally refactored:
- Lifted FORMS, FIX_CORR, bootstrap_corr_lo, model_overall_map, _load_teacher_from_val to module scope
- Extracted data pipeline into `build_oracle_pipeline()` returning `(target, reward_series, present, n_common)` — importable without running the full script
- Guarded CLI execution under `if __name__ == "__main__"`
- CLI output byte-identical to the prior version (same print lines, same JSON write)

`scripts/reward_validity_gate.py` — new standing gate entrypoint:
- Exports `run_validity_gate(form_name, form_fn?, n_boot=2000) -> GateResult`
- Exports `GateResult` dataclass: `{form, spearman_vs_target, ci_lo, ci_hi, valid, n_ckpts}`
- Imports FORMS + bootstrap_corr_lo from oracle (single source of truth — no copy)
- `valid = bool(ci_lo > 0)` — the standing gate rule (D-08.2)
- Supports custom `form_fn` callable for Plans 03/04 new candidates

Known verdicts reproduced:
- `pairwise_rank_agreement`: spearman=+0.700, ci=[+0.147, +0.935], **VALID**
- `fix_correctness_BASELINE`: spearman=-0.240, ci=[-0.872, +0.417], **INVALID**

`.planning/phases/08.2-reward-validity/08.2-GATE-RULE.md` — standing rule documentation:
- Rule: "no reward goes to GPU until oracle corr CI-lower > 0"
- Baseline verdicts table
- Usage examples (built-in + custom form_fn)
- n=11/one-run wide-CI caveat (necessary screen, not a power estimate)
- Val-held-out and content-hash-join notes
