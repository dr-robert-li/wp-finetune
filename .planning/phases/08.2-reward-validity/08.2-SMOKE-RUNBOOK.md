## 1. Why This Smoke Exists

Phase 10 ran 500 steps on faith with an INVALID reward. The post-mortem (RLEV_FINAL_REPORT.md):

| Metric | Phase 10 result |
|---|---|
| fix_correctness proxy | +0.028 (rose steadily) |
| teacher-Spearman (validated) | flat — no checkpoint beyond noise |
| wp-bench codegen (validated) | −0.049 regression |

This is textbook Goodhart: the proxy rose while both validated targets moved sideways or down.

This smoke makes the failure impossible to repeat:
- The kill-at-50 rule stops a diverging run before 250 steps, never allowing 500.
- The validated metric gate (teacher-Spearman trend, NOT fix_correctness) must be measurably moving before the run continues.
- The codegen trip-wire halts training if wp-bench drops below the v1.2 SFT bar (0.4616).
- The echo-adversary gate (≤ 0.30) prevents calibration-reward Goodhart on style rather than substance.

## 3. Gates — All Must Hold to Continue Past Step 50

### Gate 1: Within-Run Paired Teacher-Spearman TREND (THE VALIDATED METRIC)

**What it is:** At each checkpoint step t, compute teacher-Spearman (ρ_t) on a fixed held-out
set using `scripts/_rlev01_score.py`. "Within-run" means comparing the same eval set across
checkpoints of THIS run (not cross-seed, not vs warm/stale — those are Phase 10's confounded
comparisons).

**Exactly how measured:**
1. Fixed held-out set: `data/rl_probe/judge_probe_corpus.jsonl` (the same set used in offline scoring).
2. At each checkpoint: run `_rlev01_score.py` against the checkpoint to get `(judge_score, teacher_overall)` pairs across all items.
3. Compute Spearman ρ between judge scores and teacher GT per-checkpoint. Call this ρ_t.
4. Trend sign: the within-run paired teacher-Spearman series `{ρ_50, ρ_100, ...}` must be **measurably increasing** — not flat, not decreasing.
5. At the step-50 kill gate: `ρ_50 > ρ_initial` (pre-run baseline) with a positive trend direction. A flat or negative ρ_50 vs ρ_initial triggers the kill rule (Section 4).
6. Bootstrap CI (n_boot=2000): the trend must have a positive lower CI bound across the checkpoints collected so far. A CI that includes zero at step 50 is treated as flat → kill.

**What it is NOT:** The `fix_correctness` proxy (`_check_judge_fixcorr`). Phase 10 proved this metric can rise 0.028 while teacher-Spearman is flat. `fix_correctness` is explicitly FORBIDDEN as the go/no-go criterion for this smoke.

### Gate 2: Codegen Trip-Wire — RVAL-03

**What it is:** At each checkpoint, the in-run wp-bench probe (`run_codegen_probe` via `--codegen-probe-every 50`) compares the current checkpoint's codegen score against `CODEGEN_BAR_V12 = 0.4616`.

**Trigger:** If `wpbench_score < 0.4616`, the trip-wire fires. `check_codegen_tripwire` returns a non-None halt reason; `halt_reason` is set; `optim_step` is NOT called for that step; training halts.

**This gate is wired in code** (`scripts/rl_codegen_tripwire.py` → `scripts/rl_train.py` run_training_step). It does not require manual intervention; it fires automatically.

Sub-floors (from Phase 10 step-500 regression report):
- `knowledge` sub-score: KNOWLEDGE_FLOOR = 0.45
- `exec` sub-score: EXEC_FLOOR = 0.375

### Gate 3: Echo-Adversary ≤ 0.30

**What it is:** At each checkpoint read, the standard echo-adversary check (Phase 10 read protocol, `READS_TALLY.md`) must report echo ≤ 0.30.

**Context:** Phase 10's seedA held echo = 0.25 across ALL 10 reads (steps 15–500). An echo score above 0.30 indicates the judge reward is being gamed by pattern-matching without semantic content.

**Failure action:** If echo > 0.30 at any step-50 read, KILL. Do not continue to step 100.

---

## 4. Kill-at-50 Rule

**This is the single most important rule in this runbook.**

**Condition:** If, at step 50, ANY of the following holds, KILL THE RUN immediately. Do not push to 250. Do not run 500 steps.

1. Within-run paired teacher-Spearman (ρ_50) is NOT measurably higher than ρ_initial (flat or negative trend, or CI includes zero).
2. Codegen trip-wire has fired (wp-bench < 0.4616 at step 50).
3. Echo-adversary > 0.30.

**Kill action:** `kill <PID>` on the training process. The trip-wire (Gate 2) handles this automatically in code. Gates 1 and 3 require a human read at step 50.

**What "kill" means:** Stop. Do not save a final checkpoint. Do not adjust hyperparameters and relaunch. Return to Phase 08.2 planning. The reward configuration is not validated for GPU spend.

**What "not moving" means for Gate 1:** ρ_50 ≤ ρ_initial + 0.02 (a 0.02 margin of noise tolerance). Below this, the signal is indistinguishable from run-to-run variation in the probe corpus.

## 5. Go/No-Go Decision Tree

```
                    [Step 50 Read]
                         |
        ┌────────────────┴────────────────┐
        | All 3 gates PASS?                |
        └────────────────┬────────────────┘
               YES       |        NO
                ↓        |        ↓
         CONTINUE        |      KILL
         to step 250     |      (Section 4)
                ↓        |
         [Step 100, 150, 200: monitor]
         (codegen trip-wire auto-halts if fired)
                ↓
          [Step 250 BINDING READ — n=80 per seed]
                |
        ┌───────┴───────┐
        | PASS?          |
        └───────┬───────┘
           YES  |   NO
            ↓   |    ↓
     ESCALATE   |  KILL
     to gated   |  (do not push to 500)
     Phase 9    |
     rerun       |
```

### Step 50 Read Protocol (mirrors READS_TALLY.md)

1. Run `_rlev01_score.py` against step-50 checkpoint (held-out set, n=40 per policy).
2. Compute within-run paired teacher-Spearman (Gate 1).
3. Check codegen trip-wire result from the automated probe (Gate 2 — already wired in training).
4. Run echo-adversary check (Gate 3).
5. Record in a `SMOKE_READS_TALLY.md` (parallel to `READS_TALLY.md`).
6. Apply kill-at-50 rule.

### Step 250 Binding Read

If the run reaches step 250 without the trip-wire firing and without a manual kill:
- Increase to n=80 (40 per seed) for the binding read (Phase 10 binding gate protocol).
- Compute late-early within-run ρ: (ρ_250 − ρ_50) must be positive for BOTH seeds.
- Echo ≤ 0.30 must hold.
- Codegen must be ≥ 0.4616 on both seeds.

### What a PASS at 250 Authorizes

A PASS at step 250 authorizes **escalation to a gated Phase 9 rerun** — it does NOT auto-promote to a full 500-step run. A gated Phase 9 rerun requires:
1. A written go/no-go from the operator (not an automated decision).
2. A full RLEV-01/02 gate (wp-bench + anti-hack + judge-Spearman) on step-250 checkpoints before continuing.
3. A step budget increase proposal specifying the rationale for going beyond 250.

**No 500-step runs on faith.** The Phase 10 lesson is encoded here as a process requirement.
