# Teacher-Ceiling Measurement (2026-07-03) — the target was exhausted

**Question:** after the honest seedA2 smoke failed (teacher-Spearman never left noise), is the
recovery a better reward, or a better teacher? Decision threshold: headroom ≥ ~0.1 → reward-v2;
thin → better teacher.

## Method
Blind re-judge of the 86 val `<wp_judge>` items that carry stored teacher GT
(`_extract_gt_from_assistant` on `openai_val.jsonl` — the same 86 used by `_rlev01_score.py`'s
aligned set). Two independent passes (A, B) by current Claude via in-session Claude Code agents
(D-13 pattern, $0; 10 agents/pass, 9 items/agent; GT withheld from agents; same judge-output JSON
contract as the dataset: dims 0-10, overall_score 0-100). Reliability of the stored single-pass GT
label estimated by the classic triad/factor-loading identity `rel_GT = ρ(A,GT)·ρ(B,GT)/ρ(A,B)`;
ceiling on any policy's observable ρ vs stored GT = `sqrt(rel_GT)` (Spearman attenuation).
Artifacts: `output/rl_eval/teacher_ceiling/{out_A_*.json,out_B_*.json,ceiling.py}`,
summary `output/rl_eval/teacher_ceiling_summary.json` (gitignored; numbers reproduced here).

## Result (n = 86 aligned)
| measurement | ρ |
|---|---|
| current-Claude test-retest ρ(A,B) | **0.9025** |
| fresh pass A vs stored GT | 0.6239 |
| fresh pass B vs stored GT | 0.6204 |
| **v1.2 SFT student vs stored GT** (warm-start baseline) | **0.6243** |
| stored-GT single-label reliability (triad) | **0.4288** |
| ceiling on observable ρ(policy, stored GT) | **0.6548** — bootstrap CI [0.497, 0.767] |
| **headroom over warm-start 0.6243** | **+0.0305** — CI [−0.128, +0.143] → **THIN** |

## Interpretation (decisive)
1. **The student already matches the teacher.** Today's Claude — the strongest available judge —
   scores ρ≈0.622 against the stored labels; the v1.2 SFT student scores 0.6243. The student
   SATURATED the stored-GT target. Everything above ~0.62–0.65 is noise-chasing by construction.
2. **The seedA2 kill was inevitable.** The kill-at-50 bar (0.6443) sat essentially at the
   measurement ceiling (0.6548). No reward — however well designed — could have cleared it.
   The RL program did not fail on optimization; **the target was exhausted.**
3. **The April-era stored GT is the bottleneck**, not the models: single-pass labels from an older
   Claude carry reliability ≈0.43, while today's Claude is ≈0.90 self-consistent on the same items.
4. **Recovery = better teacher labels.** Two-pass-averaged fresh labels (Spearman–Brown:
   2·0.90/(1+0.90) ≈ 0.95 reliability) push the ceiling to ≈ sqrt(0.95) ≈ 0.975 — real headroom
   reopens for BOTH continued SFT and any future RL. Re-labeling ~700 train + ~120 val items with
   Claude Code agents is $0 and one session of work.
