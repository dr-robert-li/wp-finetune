A complete planning document specifying the smoke that makes Phase 10's failure impossible to repeat. Sections:

6. **Go/no-go decision tree** — step 50 -> 250 (if all gates pass) -> gated Phase 9 rerun (if binding read passes). Step-250 PASS authorizes escalation ONLY — no auto-promote to 500.

- **Null-selected fallback:** when `selected` is null (the actual file state from Plan 04), filters `valid==true` rows, picks `max(ci_lo)`, first-occurrence tiebreak -> `hybrid @ calib_weight=0.8`.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Python heredoc `__file__` resolves to `<stdin>` when called via bash heredoc**

- **Found during:** Task 2 first-run verification
- **Issue:** The embedded Python block used `os.path.dirname(os.path.abspath(__file__))` to find the repo root, but `__file__` is `<stdin>` in a bash heredoc context — causing a `FileNotFoundError` for `sweep_results.json`.
- **Fix:** Passed `SWEEP_RESULTS_PATH` as an environment variable (`SWEEP_RESULTS_PATH="$SWEEP_RESULTS" "$PYTHON" - <<'PYEOF'`) and read it via `os.environ.get("SWEEP_RESULTS_PATH", ...)` inside the heredoc.
- **Files modified:** `scripts/launch_validated_smoke.sh`
- **Commit:** 4af3267 (same task commit, fixed before staging)
