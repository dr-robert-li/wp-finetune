# Teacher Re-Labeling Protocol (2026-07-03) — evidence-based, $0, Claude Code agents

**Trigger:** `08.2-TEACHER-CEILING.md` — stored GT reliability 0.43, ceiling 0.655, student
saturated at 0.6243. Recovery = better labels, not better reward.
**Evidence base:** deep-research run wf_2e1ad30f-f07 (24 sources, 25 claims adversarially
verified 3-vote: 22 confirmed / 3 refuted). Key citations inline.

## Non-negotiables the research forces (vs the naive plan)

1. **0.90 self-consistency is NOT validity** (consistency-bias paradox, arXiv 2606.19544:
   judges with 0.99 test-retest carry severe bias; raw agreement overstates chance-corrected
   agreement by 33–41 pts). QC must report **Cohen's κ / Krippendorff's α** on verdicts and run
   explicit bias audits — repeat-pass agreement alone is insufficient.
2. **Pointwise absolute scoring stays the primary label** for our 0–100 target: pairwise flips
   ~35% under distractor manipulation vs 9% for absolute (arXiv 2504.14716). Pairwise+Bradley-
   Terry is an OPTIONAL ordinal sharpener on val only (CodeJudgeBench 2507.10535 shows pairwise
   discriminates better on code, but it yields rankings not scalars, is O(n·m²), and one refuted
   claim warns pairwise "winners" may be spurious).
3. **Multi-pass averaging is the reliability mechanism** — variance shrinks ~1/M (generalizability
   theory, arXiv 2604.11581). The "~11 passes to converge" figure was REFUTED (0-3) — no
   validated M exists; choose M from Spearman–Brown math on OUR measured single-pass reliability.
4. **Active allocation beats uniform passes** — re-labeling prioritized by disagreement/difficulty
   corrects labels up to 4× more efficiently (Nature Commun. 2022, Bernhardt et al.).
5. **No debate protocols** — multi-agent debate AMPLIFIES judge bias after round 1 (2505.19477).
   No big panels — correlated errors mean 9 judges ≈ 2.2 effective votes; saturation at ~2–3.
6. **Judge the full, unstripped code** (keep comments), use reasoning-before-scoring, and freeze
   a cue-free rubric prompt: injected sentiment cues flipped a code judge from 67%→93%/→13% on
   unchanged code (Bias in the Loop, 2604.16790); style/format bias dominates position bias.
7. **Re-label val with the same rigor as train** — label noise distorts evaluation and model
   selection, not just training (label-convergence, 2409.09412).

## Sample-size math (from OUR measured numbers)

Single-pass reliability of current Claude on this task = measured test-retest ρ(A,B) = **0.9025**.
Spearman–Brown for M averaged passes: rel_M = M·r/(1+(M−1)·r):
| M | rel_M | ceiling √rel_M |
|---|---|---|
| 1 | 0.902 | 0.950 |
| 2 | 0.949 | 0.974 |
| **3** | **0.965** | **0.983** |
| 5 | 0.979 | 0.989 |

**M=3 base passes** already lift the ceiling from 0.655 → ~0.98. Diminishing returns beyond;
spend extra passes actively (step 4) instead of uniformly.

## Protocol (executable now, in-session agents, $0 — D-13 pattern)

**0. Freeze the instrument.** One rubric prompt file, versioned in-repo: 9 dimensions with
0–10 anchors (what a 2 / 5 / 8 looks like per dimension), the exact `<judge_output>` JSON
contract (dims 0–10, omit-if-N/A, verdict, overall_score 0–100), NO sentiment/leading cues,
code presented verbatim (comments retained). Never edit mid-campaign — format sensitivity is a
documented drift source.

**1. Pilot = val re-label (120 items; 86 already have 2 passes banked).** Run to M=3
(1 more pass for the 86, 3 passes for the ~34 GT-less val items). Aggregate: **median** of
overall_score across passes; per-dim median. QC gates before ANY train spend:
   - split-half reliability of aggregated labels ≥ 0.95 (Spearman–Brown projected);
   - κ on PASS/FAIL verdicts across passes ≥ 0.6 (chance-corrected, not raw %);
   - bias audit: (a) score↔code-length Spearman |ρ| < 0.2 (verbosity proxy);
     (b) reformat-probe on 10 items (whitespace/comment-layout shuffle → median |Δoverall| ≤ 5);
   - validity anchor: new labels vs deterministic `_extract_verifiable_signals` rubric ρ > 0.4,
     and Dr. Li spot-checks 10 items (5 highest-disagreement, 5 random).

**2. Sentinel drift control.** 5 fixed sentinel items embedded in every agent batch; a batch
whose sentinel scores deviate >10 pts from the campaign median is re-run (batch-effect monitor).

**3. Train re-label (~700 wp_judge items × 3 passes ≈ 2,100 judgments ≈ 78 agent-batches of 9×3).**
Same frozen instrument, batches randomized (no source-file ordering), 10 parallel agents per wave.

**4. Active second wave.** Items with pass-disagreement (overall range > 15 pts OR verdict split)
get +2 passes; re-aggregate. Expect ~15–25% of items (the 4× efficiency lever).

**5. Optional ordinal sharpener (val only, if RL reopens).** Round-robin pairwise within
±10-pt score bands, BOTH orders (AB+BA, inconsistency = tie), Bradley–Terry re-rank, use to
break ties in the 0–100 labels. Skip if SFT-only.

**6. Rebuild + re-measure.** Rebuild `judge_gt_sidecar.jsonl` + val GT from new labels
(same `normalized_code_hash` join). Re-run `_rlev01_score.py`-style eval: student vs NEW labels.
The measured gap is the REAL learnable signal:
   - gap ≥ ~0.15 → continued SFT on re-labeled data first (cheapest), RL reconsideration after;
   - gap < ~0.05 → the student is genuinely near-teacher; improvement requires a stronger
     teacher signal than labels (e.g. defect-injection synthetic GT), not more training.

## Cost
All LLM work in-session Claude Code agents ($0, subscription). Pilot ≈ 15 agent-batches;
train wave ≈ 78; active wave ≈ 20. Comparable to this session's ceiling measurement (~20 batches
in ~4 min wall-clock per wave of 10).

## Refuted ideas (do not resurrect)
- "~11 majority-vote passes needed" (0-3 refuted) — use Spearman–Brown on measured r instead.
- "Judge diversity dominates rubric quality" (1-2) — invest in the rubric, not model zoo.
- Debate protocols (bias-amplifying), large panels (correlated errors), pairwise as primary
  label for scalar targets (distractor-fragile).
