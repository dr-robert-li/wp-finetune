<objective>
Prove BASE-04: the VL checkpoint's merge path works end to end — a LoRA adapter targeting
model.language_model.* is merged by merge_adapter.py, the merged model serves via vLLM
--language-model-only, and a real generation is coherent AND observably adapter-influenced.
The dual key-prefix silent-partial-load risk (merge-time model.language_model.* vs serve-time
--language-model-only remap) is the whole point — a merge that exits 0 and a serve that boots
healthy can still be a silent partial-load, so the acceptance test is a base-vs-merged output
DIFF, not two independent green checks.

Purpose: De-risk the SFT→merge→serve chain before Phase 21 spends real Tinker/GPU budget on it.
Output: prefix-aware merge_adapter.py, a probe-adapter builder, the round-trip smoke + receipt.

Runs on the single GB10 GPU / unified pool; placed at wave 4 so no other 67 GiB model is
resident (merge loads the base on CPU and exits before the vLLM serve step boots — do not run
both memory-heavy steps concurrently). Depends on 20-03 for the serving harness.
</objective>
