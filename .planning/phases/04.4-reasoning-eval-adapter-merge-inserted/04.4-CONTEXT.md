<decisions>
## Implementation Decisions

### Eval architecture (carries the 4.3 SUMMARY blocker)

- **D-01:** **Defer the load/serve path choice to the phase researcher.** ckpt-72's `adapter_config.json` has `target_parameters: [mlp.experts.gate_up_proj, mlp.experts.down_proj, down_proj]` — an Unsloth-trained parameter-level MoE-expert LoRA. Three candidate paths (researcher must produce evidence-backed recommendation):
  1. **Unsloth merge → vLLM serve bf16** — `FastLanguageModel.save_pretrained_merged(save_method='merged_16bit')` bakes target_parameters into expert weights, then `serve_30_70_vllm.sh` pattern (GPU_MEM_UTIL=0.55 worked previously). Bias toward this since wp-bench needs HTTP serving.
  2. **Unsloth FastLanguageModel in-process bf16** — load merged base + ckpt-72 adapter; close heavy desktop apps + raise GB10 pre-flight floor to 70 GiB. Works for eval_judge/eval_gen but wp-bench needs an HTTP path → mismatch.
  3. **GGUF Q8 via llama.cpp** — one-time convert merged-with-adapter to GGUF Q8 (persistent artifact, fast subsequent loads, Q8 preserves MoE routing far better than int4). Verify llama.cpp Q8 support for target_parameters-merged Qwen3-MoE.

- **D-02:** **Hard constraint researcher must satisfy:** chosen path MUST (a) serve via OpenAI-compatible HTTP (wp-bench HARD GATE is the showstopper), AND (b) preserve target_parameters MoE LoRA semantics (verify by smoke test: 3 prompts on ckpt-72 → coherent output → output differs measurably from base 30_70-merged). If a path can't satisfy both, eliminate it. No path locked until smoke passes.

### Baseline definition + re-eval policy (REVL-01/02/04)

- **D-03:** **Researcher audits stored Phase 4 30_70 baseline numbers + recommends re-eval scope.** Sources to audit: `output/triage_decision.md`, Phase 4 eval JSON (per-dimension Spearman, PHPCS pass rate, dimension means). Decision criteria for researcher's recommendation:
  - If the new harness (vLLM-merged bf16) differs materially from Phase 4 harness (likely LoRA-on-base via different serving path), stored numbers are NOT apples-to-apples → recommend fresh re-eval of 30_70-merged on the same harness.
  - wp-bench was SKIPPED in Phase 4 triage — fresh 30_70 baseline IS required for wp-bench regardless (no stored numbers).
  - Default toward re-eval if any harness change is non-trivial; trust stored numbers only if they were produced on a path equivalent to the chosen 4.4 path.
- **D-03b:** Triage-script HUMAN_OVERRIDE sentinel: the auto-regen of `output/triage_decision.md` clobbered the 30_70 human override into "NO_SURVIVORS" (see 4.3-CONTEXT carried follow-up). Add a `HUMAN_OVERRIDE` sentinel / do-not-regenerate guard before any 4.4 work touches the triage decision file. Do NOT commit the current clobbered copy (`git restore output/triage_decision.md` if it interferes).

### REVL-05 human review modality

- **D-04:** **~20 mixed CoT + CtF samples in a single markdown side-by-side file.** Format: prompt → reasoning-adapter output (parsed: reasoning chain + structured judge_output + scores) → baseline 30_70 output (same parse) → reference (where applicable). Stratified selection: 10 CoT + 10 CtF; within each stream, 4 high-confidence, 4 low-confidence (near boundary), 2 random. Render in editor/preview (no UI). Sign-off mechanism: `HUMAN_APPROVED: <timestamp>` sentinel line in `output/v1.2_human_review.md`; absence of sentinel = merge blocked.

### Regression / failure policy

- **D-05:** **Hard block v1.2 on any HARD-gate failure** (Spearman drop, PHPCS regression > 2pp, wp-bench fail). No "close enough" pass. Options on failure:
  1. **Iterate:** more reasoning data (CoT backfill toward original D-03 target ≈ 700 examples), adjusted LR, more epochs, or fresh-LoRA-on-different-base. Requires re-opening Phase 4.3 or earlier.
  2. **Abandon reasoning track:** ship v1 30_70 as v1.2-final (no reasoning enhancement), archive ckpt-72 unmerged, move to v2 RL.
  Decision between (1) and (2) is made at the failure point, NOT pre-committed here. Reasoning-length-distribution and classification-matrix metrics are SOFT (flag-only, not block); Claude-evaluator reasoning quality (REVL-03) and fix correctness (REVL-06) thresholds set at planning time.
