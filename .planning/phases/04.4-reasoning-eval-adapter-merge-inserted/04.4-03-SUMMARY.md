## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing Critical] fidelity_gate_v3.assert_served_identity: added optional served_model_name param**
- **Found during:** Task 1 (wiring assert_served_identity into run_eval_reasoning.py)
- **Issue:** The function hardcoded SERVED_MODEL_NAME = "wp-reasoning-v3" as the expected served name. However, boot_vllm (used in run_eval_reasoning.py) calls serve_30_70_vllm.sh which always passes --served-model-name wp-30_70 regardless of MODEL_DIR. Using the hardcoded name would cause a false-fail on the identity check — the fingerprint (merge_report + shards) still proves v3, but the served name differs.
- **Fix:** Added `served_model_name: str = SERVED_MODEL_NAME` optional param to assert_served_identity. Default unchanged = "wp-reasoning-v3" (backward compatible for callers using serve_reasoning_v3_vllm.sh). run_eval_reasoning passes "wp-30_70" explicitly.
- **Files modified:** scripts/fidelity_gate_v3.py
- **Verification:** Param is optional (default preserves existing behavior); new param is threaded into the name_ok check; docstring updated to document the dual-caller pattern
- **Committed in:** 78fba61 (Task 1 commit)
