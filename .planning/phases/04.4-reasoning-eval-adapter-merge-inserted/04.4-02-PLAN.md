<objective>
In a SINGLE vLLM serve session of the clean v4 staging model, capture every input the automated gates need: merged-endpoint judge responses (val + 24 sentinel), reasoning samples, and gen samples — then (only if the byte-identity check failed) re-bench REVL-04 in the same session before teardown.

Purpose: The 121 GiB unified-memory ceiling makes the vLLM serve a singleton resource — two serve-requiring plans would OOM, and each serve/teardown is slow and crash-prone. Mirroring the 04.3 grid's "serve once, capture everything, compute offline" pattern collapses all GPU-bound work into one wave and lets Plan 03 compute all gates offline from `--responses-jsonl` captures. D-V4-03 mandates the judge re-confirm be measured on the MERGED model (the 04.3 judge gates were on the Tinker sampler — the exact v3 failure mode this verifies against).

Output: judge_val + sentinel + reasoning + gen capture JSONLs + capture_manifest.json (+ revl04_rebench/ only if reuse_revl04==false).
</objective>
