# Phase 04.4 — Mechanism Diagnosis: Why the v4-winner merged endpoint "failed" the judge gates

**Date:** 2026-06-14
**Trigger:** Plan 03 8-gate cascade returned `automated_pass=false`. Human chose "diagnose mechanism
first" before retrain-vs-waiver. This file is the diagnosis.

## Question

The clean v4-winner merge passed codegen (REVL-04 wp-bench **0.4603 ≥ 0.4537**, reproduced the grid
exactly) and the merge anchors all certified (tensor byte-exact, fp32-control, forward router-invariant
— faithful weights). Yet the merged-endpoint judge gates "failed":

- REVL-01A Spearman (canonical GT): **0.240 < 0.263 bar** → HARD FAIL
- confusion false_FAIL_on_teacherPASS: **0.468 > 0.403 bar** → HARD FAIL
- REVL-02 PHPCS 0.9412 < 0.98 (1/17, small-N, non-binding)

Grid (Tinker sampler, r32-rp30 winner) PASSED the same gates: Spearman 0.294, false_FAIL 0.387.
**Why does the same r32-rp30 MoE-only adapter pass via the Tinker sampler but "fail" via the bf16
merged vLLM endpoint?**

## Method

Both judge runs wrote per-pair dumps with **identical schema, same canonical GT, same val indices**
(`revl01a_judge_raw.pairs.jsonl` = merged; `judge_results.pairs.jsonl` = grid). Only `model_overall`
differs (grid sampler vs merged endpoint). Apples-to-apples. Bootstrap (10k resamples, seed 20260614):

| Metric | Merged | 95% CI | P(fail bar) | Grid | Grid P(fail) |
|--------|--------|--------|-------------|------|--------------|
| REVL-01A Spearman | 0.240 | **[0.061, 0.410]** | 0.60 | 0.294 [0.128,0.450] | 0.36 |
| confusion false_FAIL | ~0.40 | **[0.265, 0.537]** | 0.48 | ~0.32 [0.196,0.452] | 0.11 |
| **Paired Δρ (merged−grid)** | **−0.059** | **[−0.169, +0.052]** | — (includes 0) | — | — |

Supporting:
- REVL-01B (vs **teacher** scores) **rose**: merged 0.504 vs grid 0.476 — ranking skill not lost.
- Mean `model_overall`: merged **58.1** vs grid **61.6** → **−3.6 pt strictness offset**.
- confusion sample sizes: teacherPASS n=50, teacherFAIL n=33 (false_FAIL CI ≈ ±0.13).

## Findings

**The mechanism, stated precisely (both legs tested): a SIGNIFICANT rank-preserving calibration
offset, with a NON-significant ranking change.**

1. **Calibration shift — SIGNIFICANT.** Paired bootstrap of the mean `model_overall` difference:
   **−3.58 pt, SE 1.25, ~2.9 SE, 95% CI [−6.09, −1.24], P(offset<0)=0.999.** The merged endpoint
   scores ~3.6 pts stricter than the Tinker sampler — a real, significant downward calibration shift.

2. **Ranking change — NOT significant.** Paired Spearman difference CI **[−0.169, +0.052] includes
   zero** (directional lean P(merged<grid)=0.85 — a mild worsening that does NOT clear significance).
   REVL-01B (vs teacher) actually *rose* (0.504 vs 0.476). So judge *ranking skill* is statistically
   indistinguishable from the grid winner that passed. **"Not significantly distinguishable" — not
   "unchanged":** there is an 0.85 directional lean; the gate simply cannot confidently separate this
   model from the passing grid winner.

3. **Both HARD fails trace to the ONE significant offset, and both land inside the noise band.** The
   −3.6pt downward shift mechanically pushes both threshold-based gates the same way: lower scores →
   weaker canonical-Spearman alignment AND more sub-70 verdicts → higher false_FAIL. They "corroborate"
   only because they share this single cause — not two independent degradations.
   - REVL-01A: merged 0.240, 95% CI **[0.061, 0.410] spans the 0.263 bar** (P(fail)=0.60). Lead leg.
   - confusion false_FAIL: official 0.468 over ~50 teacher-PASS cases sits ~0.9 SE above the 0.403 bar
     (~0.8 one-sided — NOT a coin-flip, but its CI still spans the bar; weaker leg, demoted). (A
     simple-threshold reconstruction undershoots at 0.40; the official policy-based 0.468 is authoritative.)
   - Grid "passes" were themselves only ~0.64–0.89 confident. At n≈120 (Spearman SE≈0.09) / n=50, the
     absolute point-bars sit inside the CI for BOTH models.

4. **Offset mechanism:** bf16-merged inference vs Tinker's runtime LoRA application produces marginally
   different logits → the judge's absolute scores compress slightly downward. Real inference-path
   effect, touching **calibration (absolute scores), not ranking** — and a training run will not
   reliably remove a bf16-merge inference artifact.

5. **The CIs cannot be tightened by re-measuring.** Judge decode is greedy (temperature=0.0), so
   re-judging the same prompts yields identical scores. The val set is exhausted (141 rows, 121
   judge-tasks, all used). Only MORE annotated judge prompts would narrow the bands — none exist.
   "Re-measure for statistical power" is therefore a dead end, not an option.

## Implications for the decision (the decision stays the human's)

- **A mandatory retrain is not justified by the evidence**, and is likely low-value: there is no
  confident ranking-skill loss to fix, and the one significant effect (−3.6pt calibration) is a
  bf16-merge inference artifact a training run will not reliably remove. A ~6h iteration would chase a
  noise-band gate crossing.
- **But this is NOT "the model is fine, ship it."** A real, significant (if small) rank-preserving
  calibration offset DOES ship with this model into Phase 7+. The honest statement is: the gate cannot
  confidently distinguish the v4-winner from the grid winner that passed (0.85 directional lean toward
  mild worsening, not significant).
- **The offset is calibration, so it is cheaply correctable at INFERENCE time** (verdict-policy
  threshold / a small score recalibration to undo the −3.6pt shift) — a more targeted, far cheaper fix
  than retraining IF the offset is judged to matter for the Phase 7 reward signal.
- **On gate design:** frame as "the gate cannot distinguish this model from the passing grid winner,"
  NOT "lower the bar" (adjusting a bar after seeing the result is goalpost-moving). A CI-aware gate
  (require bootstrap CI-lower ≥ bar) would treat grid and merged consistently — a forward-looking note,
  not a retroactive rescue of this result.

### Decision options the human now weighs (diagnosis as tiebreaker)
1. **Waiver → run Wave 4** (REVL-05 human review + promote). Evidence-backed: ranking skill
   statistically intact; the fail is a noise-band crossing from a small calibration offset. Ships the
   −3.6pt offset into Phase 7+ (human-eyes review of merged samples is the safeguard).
2. **Recalibrate at inference, then re-gate** (cheap, no retrain): apply a score/threshold correction
   for the −3.6pt offset, recompute the offline cascade. Targets the actual measured effect.
3. **Retrain (re-open 04.3)**: lowest expected value per this diagnosis — no confident skill loss, and
   the offset is a merge-inference artifact, not a training deficiency.
