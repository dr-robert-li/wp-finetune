# Phase 04.4 — Mechanism Diagnosis: Why the v4-winner merged endpoint "failed" the judge gates

**Date:** 2026-06-14
**Trigger:** Plan 03 8-gate cascade returned `automated_pass=false`. Human chose "diagnose mechanism
first" before retrain-vs-waiver. This file is the diagnosis.

## Question

The clean v4-winner merge passed codegen (REVL-04 wp-bench **0.4603 ≥ 0.4537**, reproduced the grid
exactly) and the merge anchors all certified (tensor byte-exact, fp32-control, forward router-invariant
— faithful weights). Yet the merged-endpoint judge gates "failed":

- REVL-01A Spearman (canonical GT): **0.240 < 0.263 bar** → HARD FAIL
- confusion false_FAIL_on_teacherPASS: **0.468 > 0.403 bar** → HARD FAIL
- REVL-02 PHPCS 0.9412 < 0.98 (1/17, small-N, non-binding)

Grid (Tinker sampler, r32-rp30 winner) PASSED the same gates: Spearman 0.294, false_FAIL 0.387.
**Why does the same r32-rp30 MoE-only adapter pass via the Tinker sampler but "fail" via the bf16
merged vLLM endpoint?**

## Method

Both judge runs wrote per-pair dumps with **identical schema, same canonical GT, same val indices**
(`revl01a_judge_raw.pairs.jsonl` = merged; `judge_results.pairs.jsonl` = grid). Only `model_overall`
differs (grid sampler vs merged endpoint). Apples-to-apples. Bootstrap (10k resamples, seed 20260614):

| Metric | Merged | 95% CI | P(fail bar) | Grid | Grid P(fail) |
|--------|--------|--------|-------------|------|--------------|
| REVL-01A Spearman | 0.240 | **[0.061, 0.410]** | 0.60 | 0.294 [0.128,0.450] | 0.36 |
| confusion false_FAIL | ~0.40 | **[0.265, 0.537]** | 0.48 | ~0.32 [0.196,0.452] | 0.11 |
| **Paired Δρ (merged−grid)** | **−0.059** | **[−0.169, +0.052]** | — (includes 0) | — | — |

Supporting:
- REVL-01B (vs **teacher** scores) **rose**: merged 0.504 vs grid 0.476 — ranking skill not lost.
- Mean `model_overall`: merged **58.1** vs grid **61.6** → **−3.6 pt strictness offset**.
- confusion sample sizes: teacherPASS n=50, teacherFAIL n=33 (false_FAIL CI ≈ ±0.13).

## Findings

1. **No statistically significant judge degradation.** The paired Spearman difference CI
   **[−0.169, +0.052] includes zero**. Merged and grid are statistically indistinguishable on judge
   ranking skill. REVL-01B (teacher-corr) actually improved.

2. **Both "HARD fails" are noise-band crossings, not confident fails.** The merged REVL-01A 95% CI
   [0.061, 0.410] **spans** the 0.263 bar (P(fail)=0.60 — barely a coin-flip). The merged confusion
   false_FAIL CI [0.265, 0.537] **spans** the 0.403 bar (P(fail)=0.48). The grid "passes" were
   themselves only 0.64–0.89 confident. At n≈120 (Spearman SE≈0.09) and n=50 (false_FAIL),
   absolute point-bars sit *inside* the noise band — both grid-pass and merged-fail are the same
   behavior measured twice.

3. **One root cause: a small calibration offset, not skill loss.** The merged endpoint scores ~3.6 pts
   stricter than the Tinker sampler. This single offset mechanically nudges BOTH threshold-based gates
   the "wrong" way at once (lower scores → weaker canonical-Spearman alignment + more sub-70 verdicts →
   higher false_FAIL). The two gates "corroborate" only because they share this one cause — they are
   not two independent degradations. Spearman *rank* correlation (skill) is preserved.

4. **Likely mechanism of the offset:** bf16-merged inference vs Tinker's runtime LoRA application
   produces marginally different logits → the judge's absolute scores compress slightly downward. This
   is a real inference-path effect but it touches **calibration (absolute scores), not ranking**, and
   is small enough that every gate crossing is within sampling noise.

## Implications for the decision

- **Retrain is NOT indicated.** There is no ranking-skill degradation to fix. The candidate causal
  effect is a ~3.6pt inference-path calibration offset — a training run will not reliably remove a
  bf16-merge inference artifact, and chasing a noise-band gate crossing wastes a ~6h iteration.
- **The gate design is the real weakness.** Absolute point-bars at n≈120 / n=50 are statistically
  underpowered: both PASS (grid) and FAIL (merged) live in the same CI. A CI-aware gate (require
  bootstrap CI-lower ≥ bar, or enlarge the eval N) would treat grid and merged consistently.
- **If the −3.6pt offset matters downstream**, it is correctable at **inference time** (verdict-policy
  threshold / score calibration), not by retraining.
- **Waiver → Wave 4 is now evidence-backed**: judge ranking skill is statistically intact; the
  automated fail is a noise-band crossing driven by a small calibration offset, not the merge-induced
  judge regression D-V4-03 was guarding against.
