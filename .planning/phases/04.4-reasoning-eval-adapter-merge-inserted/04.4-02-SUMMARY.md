## Deviations from Plan

### Execution Mode Deviation

**Orchestrator-driven background execution (structural, not a rule 1-3 auto-fix)**

- **Context:** The plan was designed for a subagent executor to run serve/capture/rebench sequentially. However, subagent children die on context-yield — long-running GPU jobs (vLLM serve ~2 min startup, capture ~20 min, rebench ~2.7 h) cannot survive a subagent context switch.
- **Resolution:** The orchestrator dispatched all compute via background Bash (survives context-yield). This executor performed bookkeeping only: verified artifacts exist, wrote the final disposition artifact, committed, and updated planning state.
- **Impact:** All planned artifacts are present and verified. Functionally identical outcome; only the execution locus changed (orchestrator vs subagent).
- **Pattern established:** Long GPU/vLLM jobs in wp-finetune phases MUST be orchestrator-dispatched (background Bash). Subagent executors handle bookkeeping + commit after compute completes.
