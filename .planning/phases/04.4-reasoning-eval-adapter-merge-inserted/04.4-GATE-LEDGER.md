## REVL-06 ‚Äî disposition: N/A (Option C), recorded 2026-06-02

**Basis:** 0 `<corrected_code>` emissions across 478 eval rows (full cot+ctf set, train+val).
The Phase-4.3 model is judge-only ‚Äî trained on `<wp_judge>`‚Üí`<judge_output>` reasoning, NOT
on CtF fix generation. It was never trained to emit `<corrected_code>`. A `<corrected_code>`
lint/PHPCS/security gate therefore has an empty parseable set ‚Üí vacuous; running it would
either wave the merge through silently or HARD-fail on a vacuous set ‚Äî both meaningless.

**Why not the alternatives:**
- Option A (lint teacher-authored `<corrected_code>`) tests the training-data pipeline, not
  the model ‚Äî a data-QA concern (Phase 4.1/4.2), not a model-quality eval gate.
- Option B (re-prompt 35 ctf snippets with fix framing) tests UNTRAINED behavior; high
  unparseable rate is the expected outcome, not a quality signal ‚Äî and costs a 2nd vLLM boot.

**Coverage:** fix/execution correctness is already gated by **REVL-04** (wp-bench execution
tests, PASS 0.4616 ‚â• 0.4286). The gate slot is RETIRED, not vacuously passed.

**Downstream implication:** W5-01 (REVL-05 human review) becomes **CoT-only** (10 samples,
not 20 mixed). Drop the CtF stream from the review artifact rather than leave a zero-row join.
CtF capability deferred to a future milestone if CtF training data is added.

`scripts/revl06_fix_correctness.py` is NOT built (N/A ‚Äî nothing meaningful to check).

## REVL-03 ‚Äî PASS, recorded 2026-06-02

**Result:** dimension_coverage_rate **0.814 ‚â• 0.80** (n=120, full cot+ctf val set).
Supporting: score_reasoning_consistency_rate 0.968, mean_coherence 3.97.
Artifacts: `output/eval_reasoning/revl03_aggregate.json`, `revl03_claude_eval.jsonl`
(120 rows), `revl03_agent_plan.jsonl`, `reasoning_merged/captured_responses.jsonl`
(120, parseable_rate 0.992), `revl03_chunks/` (10 chunk + 10 result files).
Earlier runs superseded: 0.810 (wrong D1..D9 taxonomy, VOID); 0.852 (8-dim but @1024
capture). This 0.814 is the authoritative number (2048 capture, parseable 0.992).

**Execution method (reproducibility):**
1. `capture_reasoning_responses.py --max-tokens 2048` ‚Äî one vLLM boot (488s) on
   reasoning-merged, 120 cot+ctf prompts, temp=0. parseable_rate 0.992, 0 `<corrected_code>`.
2. `revl03_evaluator_agent.py` ‚Äî emitted 120 opaque agent prompts (8-dim taxonomy).
3. Orchestrating session dispatched **10 chunked Claude Code agents** (12 samples
   each), agentType `general-purpose` (NOT `Agent(model=sonnet, run_in_background)`
   as the docstring's ideal). Chunked-not-1-per-sample by explicit user choice. No
   Anthropic API used ‚Äî Claude Code agents only, per ROADMAP ¬ß4.4.
4. `aggregate_revl03.py` ‚Äî merged results, computed the gate.

**CRITICAL methodology correction (advisor-caught, 2026-06-02):** the FIRST run
injected a naive D1..D9 set including `error_handling` (D8) + `code_structure` (D9).
The model's ACTUAL prose rubric is 8 dims {wpcs, security, sql_safety, performance,
wp_api_usage, accessibility, code_quality, dependency_integrity} ‚Äî verified 85/85 CoT
rows; `error_handling` + `i18n` are STRUCTURALLY ABSENT from prose, `dependency_integrity`
(emitted) was missing from the naive set. First 0.810 measured taxonomy mismatch ‚Äî VOID,
inconsistent with REVL-01 `dim_map.json` (council Option 3). Dimension set now pinned in
`REVL03_DIMENSIONS` (revl03_evaluator_agent.py); denominator = per-sample key count (8).

**Capture-cap investigation (resolved):** a 1024-token first capture was suspected of
truncating reasoning (47/120 lacked `[/REASONING]`). Re-captured at 2048 (GT max=1105).
Finding: the no-close-tag rows are NOT truncated ‚Äî they are a genuine **bimodal output
mode**. ~44/120 responses are terse direct-JSON judge output (median ~264 tok, max 805,
all <1024) with no reasoning prose; the other ~76 are full prose + `[/REASONING]`. This
is the same prose/JSON bimodality documented in the W0-03 smoke gate. The 2048 recapture
only nudged close-tag 73‚Üí76 and lifted parseable 0.967‚Üí0.992, confirming the cap was not
the cause.

**Residual coverage gap (genuine model behavior, not artifact):** the ~44 terse JSON-mode
samples skip several dimensions in their (absent) explanation text, pulling coverage from
~1.0 (prose format) down. This is why 0.814 sits near the 0.80 floor ‚Äî it reflects a real
mix of thorough-prose and terse-JSON judging modes, honestly measured.
