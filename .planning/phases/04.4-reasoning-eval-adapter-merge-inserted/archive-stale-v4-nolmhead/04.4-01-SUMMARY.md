## Deviations (3)

1. **VALIDATION/RESEARCH w1/w3 "same delta" spec error** (recorded per plan instruction). The
   obsolete claim "w1/w3 shared-A ⇒ same delta per expert" is FALSE — only lora_A is shared;
   lora_B is per-expert, so delta_e DIFFERS across experts for w1/w2/w3. Tests assert deltas
   DIFFER (broadcast rejected); a `delta_e0==delta_e1` assertion would have passed a broadcast bug.

2. **Tokenizer assertion spec error.** Plan/Task-2/Task-3 said assert `tokenizer_vocab==151936`.
   That number is the *padded model embedding*; the stock tokenizer has **151669** tokens.
   Evidence `scripts/tinker_reasoning_data.py:15-16`: v3 trained on the STOCK Qwen3 tokenizer
   where `<wp_gen>`/`<wp_judge>` are PLAIN TEXT (multi-piece), not the extended tokenizer's single
   ids 151669/151670. Merge correctly saved the stock tokenizer; assertion rewritten to the
   meaningful T-0441-02 invariant (task markers tokenize as text + len 151669 + max_id < 151936).
   **Downstream impact:** plan 05 task-routing validation must expect text-pattern routing, NOT
   single special-token ids. Plan 02 serve must use the stock tokenizer.

3. **Anchor weight-recon recalibration.** First anchor run: tensor + fp32-control failed by ~1
   bf16 ulp while the independent forward anchor passed. Cause: weight anchors reconstructed
   `bf16(base + full-precision delta)` but the merge does `delta→bf16` BEFORE the bf16 add. Fixed
   to model the merge exactly → tensor residual 1.95e-3 → **0.0** (stricter); fp32 on the bf16-noise
   criterion (rms ≪ floor on all 24 rows). Anchors re-run standalone (no re-merge; staging weights
   unchanged). Same bf16-vs-fp32 recalibration the project already applied to the forward anchor.
