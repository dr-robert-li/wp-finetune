# Phase 04.4 Recalibration Summary

**One-liner:** Judge +3.58 recalibration applied (D-V4-09): sentinel safe, confusion improved 0.468→0.452 (not cleared); REVL-01A/REVL-02 rank-invariant/codegen; automated_pass still false; Phase-8 constant recorded.

## What Was Done

Per D-V4-09 and the mechanism diagnosis (`04.4-D-V4-JUDGE-MECHANISM-DIAGNOSIS.md`), the bf16-merged endpoint applies a significant −3.58 pt calibration offset vs the Tinker-sampler reference (95% CI [−6.09, −1.24], ~2.9 SE, P=0.999, n=118 pairs, seed=20260614). This is a real, significant calibration shift that does NOT reflect a ranking-skill degradation (paired Δρ CI [−0.169, +0.052] includes 0).

Recalibration was applied by re-running both threshold-based checkers with `--threshold 66.42` (equivalent of applying policy-70 threshold to scores recalibrated by +3.58):

```
python scripts/check_verdict_confusion.py \
    --responses-jsonl output/eval_reasoning_v4_winner/judge_val_responses.jsonl \
    --threshold 66.42 \
    --output output/eval_reasoning_v4_winner/confusion_result.recalibrated.json

python scripts/check_invalid_php_sentinel.py \
    --responses-jsonl output/eval_reasoning_v4_winner/sentinel_responses.jsonl \
    --threshold 66.42 \
    --output output/eval_reasoning_v4_winner/sentinel_result.recalibrated.json
```

The Phase-8 inheritable constant was recorded in `output/eval_reasoning_v4_winner/judge_recalibration.json`.

## What Recalibration Cleared

**Sentinel (confirmed safe):** policy_false_pass = 0 at threshold 66.42. The −3.58 recalibration cannot create sentinel false-passes: raw_false_pass=0 at score-threshold 60 means all 24 should-fail prompts scored well below 66.42. The sentinel was already PASSING; recalibration confirms it remains safe under the lenient-direction threshold shift.

## What Recalibration Improved But Did Not Clear

**Confusion Pareto:** false_FAIL improved from 0.468 → 0.452 (rescued 1 case in the [66.42,70) score band). Recall remains 0.845. However, false_FAIL 0.452 still exceeds the 0.403 bar.

**Root cause of residual failure:** The majority of false-FAIL cases have `model_raw="FAIL"` (the model emitted an explicit FAIL verdict token). The `_effective()` policy returns FAIL on explicit model FAIL regardless of score. This explicit-FAIL floor is not moved by a score-offset recalibration. Approximately 27/62 teacher-PASS cases received explicit FAIL tokens; only 1 additional case was in the score-boundary [66.42,70) band that recalibration could rescue.

## What Recalibration Cannot Affect

**REVL-01A Spearman (0.240, FAIL):** Spearman is a rank metric. A uniform additive score offset is mathematically rank-invariant: `spearman(score+3.58, gt) == spearman(score, gt)` (verified by the orchestrator bootstrap: 0.2401 identical before and after offset). This gate CANNOT be cleared by recalibration. It is a noise-band fail (CI [0.061, 0.410] spans the 0.263 bar; P(fail)=0.60) — meaning the gate cannot confidently distinguish the v4-winner from the passing grid winner. Residual disposition: human waiver required.

**REVL-02 PHPCS (0.9412, FAIL):** Codegen quality gate is entirely independent of judge calibration. 1/17 PHPCS failures. Binomial CI is wide at this N; the single failure is not statistically decisive. Residual disposition: human waiver required.
