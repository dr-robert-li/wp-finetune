**scripts/eval_gen.py** (EVAL-01, EVAL-03):

Importable functions:
- `extract_code_from_assistant(content: str) -> str`: Extract PHP code from markdown fences or raw content.
- `filter_wp_gen_examples(dataset_path: str) -> list[dict]`: Load JSONL, return only examples where user content contains `<wp_gen>`.
- `run_phpcs(code: str, standard: str = "WordPress") -> dict`: Write code to temp .php file, run `phpcs --standard={standard} --report=json`, parse JSON output. Return `{"errors": int, "warnings": int, "passed": bool}`.
- `run_phpcs_security(code: str) -> dict`: Run phpcs with `--sniffs=WordPress.Security.EscapeOutput,WordPress.Security.NonceVerification,WordPress.Security.ValidatedSanitizedInput,WordPress.DB.PreparedSQL`. Return same dict format.
- `compute_pass_rate(results: list[dict]) -> float`: Count passed/total.
- `compute_security_rate(results: list[dict]) -> float`: Count security-passed/total.

Main block:
1. Accept `--model-url` (default: `http://localhost:8020/v1`), `--model-name` (default: `qwen3-wp`), `--test-data` (default: `final_dataset/openai_test.jsonl`), `--output` (default: `output/eval_gen_results.json`).
2. Filter wp_gen examples from test data.
3. For each example: call vLLM via `openai.OpenAI(base_url=model_url, api_key="none")` with the user prompt. Extract code from response.
4. Run phpcs on generated code. Run phpcs_security on security-relevant subset (detect by checking if prompt mentions nonce, sanitize, escape, prepare, capability).
5. Compute pass rates. Write results JSON with: `{"phpcs_pass_rate": float, "security_pass_rate": float, "total_gen": int, "total_security": int, "details": [...]}`.

**scripts/eval_judge.py** (EVAL-02):

Importable functions:
- `parse_judge_output(response: str) -> dict`: Parse model's judge response to extract `overall_score` (int 0-100) and dimension scores.
- `compute_spearman(model_scores: list, phpcs_scores: list) -> tuple[float, float]`: Return (correlation, p_value) via `scipy.stats.spearmanr`.

Main block:
1. Accept same `--model-url`, `--model-name`, `--test-data`, `--output` args.
2. Filter wp_judge examples from test data (user content contains `<wp_judge>`).
3. For each example: extract the code being judged from the user prompt. Send to model via vLLM in wp_judge mode. Parse model's overall_score.
4. Run phpcs on the same code to get error count. Compute phpcs_score = `max(0, 100 - errors * 5)`.
5. Compute Spearman correlation. Write results JSON: `{"spearman_corr": float, "p_value": float, "total_judge": int, "details": [...]}`.

**scripts/eval_gate.py** (EVAL-05):

Importable functions:
- `check_gates(phpcs_rate: float, spearman: float, security_rate: float) -> tuple[bool, list[str]]`: Check against thresholds (0.95, 0.85, 0.98). Return (all_pass, failure_reasons).
- `load_eval_results(gen_path: str, judge_path: str) -> dict`: Load the two JSON results files, extract metrics.

Main block:
1. Accept `--gen-results` (default: `output/eval_gen_results.json`), `--judge-results` (default: `output/eval_judge_results.json`).
2. Load results, call check_gates.
3. Print pass/fail for each gate with actual values.
4. `sys.exit(0)` if all pass, `sys.exit(1)` if any fail.
