**scripts/prepare_tokenizer.py** (MODL-02, MODL-03, MODL-04):

Must expose these importable functions (for tests):
- `add_task_tokens(tokenizer) -> tokenizer`: Adds `<wp_gen>` and `<wp_judge>` as additional_special_tokens. Returns modified tokenizer.
- `mean_init_new_tokens(embedding: nn.Embedding, num_new: int)`: Computes mean of existing rows `[:-num_new]`, assigns to last `num_new` rows in-place.
- `run_smoke_test(model, tokenizer) -> bool`: Asserts each task token encodes to single ID, generates 50 tokens from a `<wp_gen>` prompt, returns True if output length > 10.

Main block (`if __name__ == "__main__"`):
1. Load tokenizer from `./models/Qwen3-30B-A3B` (or `--model-dir` arg)
2. Call `add_task_tokens(tokenizer)`
3. Load model via `AutoModelForCausalLM.from_pretrained(model_dir, torch_dtype=torch.bfloat16, device_map="auto")`
   - IMPORTANT: Use AutoModelForCausalLM here (not FastLanguageModel) because this is pre-LoRA tokenizer prep.
   - Set `attn_implementation="flash_attention_2"` if available, fallback to default.
4. Call `model.resize_token_embeddings(len(tokenizer))`
5. Call `mean_init_new_tokens(model.model.embed_tokens, num_new=2)`
6. Note: lm_head shares weights with embed_tokens in Qwen3 — no separate init needed.
7. Save extended tokenizer to `./adapters/tokenizer/`
8. Call `run_smoke_test(model, tokenizer)` — print results.
9. Print: "Tokenizer saved to ./adapters/tokenizer/ — model base weights NOT modified on disk."

DO NOT save the full model back to disk — only the tokenizer. The base model stays pristine. LoRA + modules_to_save in Plan 03-03 will handle weight training.

Run `scripts/download_model.py` to actually download the model (this will take time with 61GB). After download completes, run `scripts/prepare_tokenizer.py` to extend tokenizer and smoke test.
