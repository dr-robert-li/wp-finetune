from scripts.dgx_toolbox import get_toolbox

The script must:

1. **Load config** from `config/train_config.yaml` (same pattern as download_model.py).

2. **Download model** (call download logic or import from download_model):
   - `huggingface_hub.snapshot_download(repo_id=config["model"]["name"], local_dir=config["model"]["local_dir"], resume_download=True, ignore_patterns=["*.msgpack", "*.h5"])`
   - Skip if model dir already has safetensors files

3. **Load model with AutoModelForCausalLM** (NOT FastLanguageModel — this is pre-LoRA tokenizer setup):
   - `torch_dtype=torch.bfloat16`
   - `device_map="auto"`
   - `load_in_4bit=False` (LOCKED — no QLoRA for MoE)

4. **Extend tokenizer:**
   - `tokenizer.add_special_tokens({"additional_special_tokens": ["<wp_gen>", "<wp_judge>"]})`
   - `model.resize_token_embeddings(len(tokenizer))`

5. **Mean-initialize new embeddings:**
   ```python
   with torch.no_grad():
       mean_embedding = model.model.embed_tokens.weight[:-2].mean(dim=0)
       model.model.embed_tokens.weight[-2] = mean_embedding  # <wp_gen>
       model.model.embed_tokens.weight[-1] = mean_embedding  # <wp_judge>
   ```

6. **Save extended tokenizer:**
   - `tokenizer.save_pretrained("./adapters/tokenizer")`
   - Also save model with resized embeddings: `model.save_pretrained(config["model"]["local_dir"])` (overwrites with new vocab size)

7. **Smoke test:**
   - Assert `len(tokenizer.encode("<wp_gen>", add_special_tokens=False)) == 1`
   - Assert `len(tokenizer.encode("<wp_judge>", add_special_tokens=False)) == 1`
   - Generate 50 tokens with a `<wp_gen>` prompt, assert output length > 10
   - Print "SMOKE TEST PASSED" on success

8. **CLI entry point:**
   - `if __name__ == "__main__":` with argparse
   - `--skip-download` flag to skip model download (for re-running tokenizer extension)
   - `--smoke-only` flag to only run smoke test (for verification)
