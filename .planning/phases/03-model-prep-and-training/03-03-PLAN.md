       max_seq_length=config["model"]["max_seq_length"],

   ```python
   model = FastLanguageModel.get_peft_model(

       lora_dropout=config["lora"]["lora_dropout"],       # 0.05
       bias="none",
       use_gradient_checkpointing="unsloth",
       modules_to_save=config["lora"]["modules_to_save"], # ["embed_tokens", "lm_head"] — LOCKED

   )
   ```

e. **Dataset loading:**
   ```python
   from datasets import load_dataset
   train_dataset = load_dataset("json", data_files=str(PROJECT_ROOT / config["data"]["train_file"]), split="train")
   val_dataset = load_dataset("json", data_files=str(PROJECT_ROOT / config["data"]["val_file"]), split="train")
   ```

       train_dataset=train_dataset,
       eval_dataset=val_dataset,
       args=SFTConfig(
           output_dir=config["training"]["output_dir"],
           num_train_epochs=config["training"]["num_train_epochs"],
           per_device_train_batch_size=config["training"]["per_device_train_batch_size"],

           max_seq_length=config["model"]["max_seq_length"],
           dataset_num_proc=4,

2. Create `scripts/merge_adapter.py` — post-training merge with verification:

```python
"""Merge LoRA adapter into base model with verification roundtrip.

Strategy (defense-in-depth):
1. Adapter already saved separately by train_model.py
2. Attempt merge_and_unload()
3. Save merged model
4. Reload and verify special tokens still work
5. If verification fails, print vLLM --lora-modules fallback command

Usage:
    python -m scripts.merge_adapter
    python -m scripts.merge_adapter --adapter-dir ./adapters/qwen3-wp
"""
from scripts.dgx_toolbox import get_toolbox
```

The script MUST implement:

a. **Config loading** from `config/train_config.yaml`

b. **Load base model + adapter:**
   ```python
   from unsloth import FastLanguageModel
   model, tokenizer = FastLanguageModel.from_pretrained(
       model_name=config["model"]["local_dir"],
       max_seq_length=config["model"]["max_seq_length"],
       load_in_4bit=False,
       dtype=torch.bfloat16,
   )
   # Load the LoRA adapter
   from peft import PeftModel
   model = PeftModel.from_pretrained(model, adapter_dir)
   ```

c. **Attempt merge:**
   ```python
   merged_path = config["model"]["local_dir"] + "-merged"
   merged_model = model.merge_and_unload()
   merged_model.save_pretrained(merged_path)
   tokenizer.save_pretrained(merged_path)
   ```

d. **Verification roundtrip:**
   ```python
   from transformers import AutoTokenizer
   verify_tok = AutoTokenizer.from_pretrained(merged_path)
   wp_gen_ids = verify_tok.encode("<wp_gen>", add_special_tokens=False)
   wp_judge_ids = verify_tok.encode("<wp_judge>", add_special_tokens=False)
   assert len(wp_gen_ids) == 1, f"<wp_gen> must be single token, got {wp_gen_ids}"
   assert len(wp_judge_ids) == 1, f"<wp_judge> must be single token, got {wp_judge_ids}"
   print("MERGE VERIFICATION PASSED")
   ```

e. **Fallback on failure:**
   ```python
   except AssertionError as e:
       print(f"MERGE VERIFICATION FAILED: {e}")
       print(f"Fallback: serve adapter directly with vLLM:")
       dgx = get_toolbox()
       print(f"  vllm serve {config['model']['local_dir']} --lora-modules qwen3-wp={adapter_dir}")
       sys.exit(1)
   ```

f. **CLI:**
   - `--adapter-dir` (default: from config training.output_dir)
   - `--output-dir` (default: model local_dir + "-merged")
