**config/train_config.yaml** — Externalized hyperparameters:
```yaml
# Qwen3-30B-A3B LoRA SFT Config
model:
  name: "Qwen/Qwen3-30B-A3B"
  local_path: "./models/Qwen3-30B-A3B"
  tokenizer_path: "./adapters/tokenizer"
  max_seq_length: 4096
  load_in_4bit: false          # LOCKED: QLoRA off-limits for MoE
  dtype: "bfloat16"
  output_router_logits: true   # LOCKED: MoE load balancing monitoring

lora:
  # NOTE: TRNG-01 specifies r=64 but we use r=32 (Claude's discretion per CONTEXT.md).
  # Rationale: With 4,766 training examples and 30B params (3B active), r=32 reduces
  # overfitting risk and memory pressure on DGX Spark. Can increase to r=64 in v2
  # if underfitting is observed. lora_alpha stays at 2*r per standard practice.
  r: 32
  lora_alpha: 64               # 2x rank
  lora_dropout: 0.05
  target_modules:
    - q_proj
    - k_proj
    - v_proj
    - o_proj
    - gate_up_proj
    - down_proj
  modules_to_save:             # LOCKED: special token embeddings must train
    - embed_tokens
    - lm_head
  bias: "none"
  use_gradient_checkpointing: "unsloth"

training:
  num_train_epochs: 2
  per_device_train_batch_size: 1
  gradient_accumulation_steps: 8    # effective batch = 8
  learning_rate: 2.0e-4
  lr_scheduler_type: "cosine"
  warmup_ratio: 0.05
  bf16: true
  fp16: false
  logging_steps: 10
  eval_steps: 100
  save_steps: 200
  report_to: "wandb"
  save_total_limit: 3

data:
  train_path: "final_dataset/openai_train.jsonl"
  val_path: "final_dataset/openai_val.jsonl"
  # NOTE: TRNG-03 specifies 50/50 wp_gen/wp_judge mix but actual data is 40/60
  # (2,383 gen / 3,575 judge). This is the accepted Phase 2 outcome — the export
  # pipeline enforced 40/60 per user decision. No rebalancing applied.

output:
  adapter_dir: "./adapters/qwen3-wp"
  wandb_project: "wp-qwen3-moe"
  wandb_run_name: "sft-lora-r32-2ep"
```

**scripts/train_model.py** — Main training script:

Must expose these importable constants/functions for tests:
- `load_config(path: str) -> dict`: Load train_config.yaml.
- `TRAIN_CONFIG_PATH`: Default path to config (`"config/train_config.yaml"`).
- `get_lora_config(config: dict) -> dict`: Return the lora section from config.
- `get_training_args(config: dict) -> dict`: Return the training section.

Main block:
1. Parse args: `--config` (default: config/train_config.yaml), `--resume` (bool, resume from checkpoint).
2. Load config via `load_config()`.
3. `wandb.init(project=config["output"]["wandb_project"], name=config["output"]["wandb_run_name"])`.
4. Load model via `FastLanguageModel.from_pretrained`:
   ```python
   model, tokenizer = FastLanguageModel.from_pretrained(
       model_name=config["model"]["local_path"],
       max_seq_length=config["model"]["max_seq_length"],
       load_in_4bit=False,       # LOCKED
       dtype=torch.bfloat16,
   )
   ```
5. Set `model.config.output_router_logits = True` explicitly after loading (Pitfall 6 — belt and suspenders).
6. Load extended tokenizer from `config["model"]["tokenizer_path"]` and apply to model:
   ```python
   from transformers import AutoTokenizer
   tokenizer = AutoTokenizer.from_pretrained(config["model"]["tokenizer_path"])
   model.resize_token_embeddings(len(tokenizer))
   ```
7. Apply LoRA via `FastLanguageModel.get_peft_model` with all config values. CRITICAL: `modules_to_save=["embed_tokens", "lm_head"]`.
8. Load datasets:
   ```python
   train_dataset = load_dataset("json", data_files=config["data"]["train_path"], split="train")
   val_dataset = load_dataset("json", data_files=config["data"]["val_path"], split="train")
   ```
9. Create SFTTrainer with SFTConfig from training section. Set `dataset_text_field=None` (SFTTrainer auto-detects "messages" column in OpenAI format). Set `packing=False`.
10. If `--resume`: `trainer.train(resume_from_checkpoint=True)`. Else: `trainer.train()`.
11. Save adapter: `model.save_pretrained(config["output"]["adapter_dir"])`. Save tokenizer alongside: `tokenizer.save_pretrained(config["output"]["adapter_dir"])`.
12. Print final training stats: total steps, final loss, peak memory.
13. **DO NOT merge adapter into base model.** Per Pitfall 2 (modules_to_save merge corruption), keep adapter separate. Phase 4 handles merge with verification.

Important anti-patterns to avoid:
- NO `load_in_4bit=True` — QLoRA is off-limits for MoE
- NO `enable_thinking=True` — Qwen3 thinking mode must be off (Pitfall 3)
- NO `merge_and_unload()` — adapter stays separate (Pitfall 2)
- NO `"gate"` in target_modules — router layer must not be fine-tuned
