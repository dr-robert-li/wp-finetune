## What was built

Full reasoning-dataset assembly pipeline for the v1.2 judge-reasoning fine-tune (DGEN stream):
consistency validation → canonical template formatting → training-mix assembly → stratified
export, plus the `run-reasoning-assembly` skill wrapper.

- **Consistency validation** (`scripts/validate_reasoning_consistency.py`): batched
  ThreadPoolExecutor LLM gate (10-example batches) rejecting examples where the reasoning
  contradicts the numeric scores. Result: 365/375 consistent, 10 rejected (2.7%, genuine
  contradictions) — the hard gate Phase 4.2 required.
- **Assembly + export** (`scripts/assemble_reasoning_dataset.py`): canonical template
  formatting (CoT + CtF carry `[/REASONING]` + `<judge_output>`; replay raw), training-mix
  assembly, stratified train/val split, `metadata.json`. Reproducible `filter_reason()`
  vendor/truncation filter (protobuf/GPBUtil/SDK-namespace regex + brace-imbalance>2).
- **Skill**: `.claude/skills/wp-finetune:run-reasoning-assembly/SKILL.md` (thin wrapper;
  decision logic stays in the scripts).
