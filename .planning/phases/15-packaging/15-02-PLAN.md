# Phase 15-02 — PKG-03 real execution: Q8 GGUF on the single-seed model

**Phase:** 15 — Packaging (PKG-03 follow-up)
**Depends on:** 15-01 (Gate 1/2 decision; Q8 was the recommended ship tier, pending toolchain)
**Requirement:** PKG-03 (advance from pre-registered to measured)

## Goal

Actually produce and evaluate a Q8 GGUF of the single-seed shipping model, closing the PKG-03 Q8 tier from
"pending toolchain" to a measured result. Target: the single-seed v1.3 judge
(`models/_staging/qwen3-30b-wp-v1.3-merged`, seed s1), since that is the model whose rho is the sensitive
metric and the one the "single seed" fallback names.

## Toolchain (provisioned this run)

- `pip install gguf`; `llama.cpp` cloned to `~/llama.cpp`, built with `-DGGML_CUDA=ON` (GB10 aarch64 + CUDA 13).
- Quantization via `convert_hf_to_gguf.py --outtype q8_0` directly (Q8_0 needs no separate llama-quantize
  pass and skips the 57 GB f16 intermediate).

## Steps

1. Convert bf16 HF -> Q8_0 GGUF. Record size (expect ~32.5 GB vs 57 GB bf16). [DONE in-run]
2. Serve the Q8 GGUF via `llama-server` (OpenAI-compatible API).
3. Capture judge responses on the val set:
   `python -m scripts.sieve_capture_judge_http --base-url http://localhost:PORT/v1 --model q8 --dataset data/reasoning_dataset/openai_val.jsonl --out output/packaging/q8_eval/judge_responses.jsonl`
4. Score rho: `python scripts/relabel/eval_relabel.py output/packaging/q8_eval/judge_responses.jsonl`.
   Same val set, same labels (`output/relabel/val_labels_v1.json`), same harness as the bf16 measurement,
   so Q8 rho is directly comparable to the bf16 single-seed s1 rho 0.8017.

## Eval arms (all served through llama.cpp for engine consistency)

Added per user: compare against the foundation model. To isolate the quantization effect cleanly, all
judge-rho arms run through the same engine (llama.cpp / llama-server), same val set, same labels:

1. **Foundation** — Qwen3-30B-A3B base, bf16 GGUF. Shows what the fine-tune added (base has no wp_judge
   training; expect low rho / parse).
2. **Shipping bf16** — v1.3 judge s1, bf16 GGUF. Engine-consistent baseline for the Q8 delta.
3. **Shipping Q8** — v1.3 judge s1, Q8_0 GGUF. The quantized ship candidate.

Cross-reference: the vLLM-measured bf16 s1 rho 0.8017 (different engine) from `optimal_k.json`.

## Gate (PKG-03 ±2pp)

- Q8 judge rho within 2pp of the bf16 s1 baseline 0.8017 (i.e. >= ~0.7817, and clearing the s1 floor 0.7497).
- Coherence: judge output parseable (NOT the degenerate collapse seen at uniform 4-bit nf4). This is the
  decisive contrast — Q8 must prove 8-bit does not break the MoE router the way 4-bit did.

## Success criteria

- Real Q8 GGUF artifact on disk with recorded size and % reduction.
- Measured Q8 rho vs bf16, within band -> Q8 tier flips to PASS in `pkg03_quantization_ladder.json`.
- Model card + JOURNAL + STATE updated; committed and pushed as dr-robert-li.

## Out of scope

Q6/Q5/Q4 tiers (Q8 is the safe ship target; lower tiers remain pre-registered). Gen-model quant (same
recipe applies; judge is the sensitive-metric proof).
