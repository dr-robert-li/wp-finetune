# Phase 15-01 — Packaging (cascading compression gates)

**Phase:** 15 — Packaging
**Depends on:** Phase 14 (re-confirmation PASS)
**Requirements:** PKG-01, PKG-02, PKG-03, PKG-04, PKG-05

## Goal

Run the cascading compression gates on the shipping pair. Since pruning shipped nothing, quantization is
the only size lever. Produce the bf16 baseline, a Gate 2 quantization decision, the PKG-03 ladder with a
±2pp stop rule, the HuggingFace model card with full lineage, and E2E validation.

## Approach decision (honest scope)

The quantization toolchain (autoawq / llmcompressor / llama.cpp) is not installed in this environment,
and the project has always quantized/served through DGX vLLM containers because a local 30B load hits the
unified-memory wall. So:

- Gate 1, Gate 2, PKG-04 model card, and bf16 PKG-05 are **fully executed** (real data, reused measured
  numbers, real prior E2E validation).
- The Q8/Q6/Q5 tier measurements are **pre-registered with a turnkey recipe** and marked pending toolchain.
  Q4 uniform nf4 is recorded as a **measured FAIL** (Phase 4.3 RTRN-04 router collapse). No unrun eval
  numbers are fabricated.

## Tasks

1. **Gate 1 (PKG-01):** record bf16 baseline (size, speed, 9-dim quality). Reuse Phase 14. ->
   `output/packaging/gate1_bf16_baseline.json`
2. **Gate 2 (PKG-02):** quantization decision. Deployment constraint (121 GB host vs 114/228 GB pair) ->
   warranted; uniform nf4 excluded by prior collapse; start Q8, activation-aware below. ->
   `output/packaging/gate2_quantization_decision.md`
3. **PKG-03:** ladder + ±2pp stop rule + measured Q4-nf4 FAIL + turnkey recipe. ->
   `output/packaging/pkg03_quantization_ladder.json`, `scripts/run_packaging_recipe.md`
4. **PKG-04:** HuggingFace model card with full lineage (base -> RL rejected -> Sieve full -> merge ->
   AIMER/REAP no_winner -> quantization), eval at each gate, usage for both task tokens. ->
   `output/packaging/MODEL_CARD.md`
5. **PKG-05:** E2E validation. bf16 shipped format validated (prior vLLM postmerge: gen 10/10, judge 10/10,
   routing 20/20); quantized tier pending. -> `output/packaging/pkg05_e2e_validation.json`
6. Close: SUMMARY, VERIFICATION, STATE + ROADMAP + REQUIREMENTS, commit.

## Success criteria (goal-backward)

- Gate 1 baseline recorded with size + quality. ✅
- Gate 2 decision documented with reasoning. ✅
- PKG-03 incremental ladder + stop rule defined; every reported result is real (Q4-nf4 FAIL measured; higher
  tiers pending, not faked). ✅
- Model card documents full compression lineage incl. AIMER/REAP no_winner + both task-token usage. ✅
- E2E confirms both tokens produce correct output on the shipped bf16 format. ✅

## Out of scope

Running the Q8/Q6/Q5 quant sweep (toolchain not provisioned here; turnkey recipe left). HF upload push
(model card ready; the `huggingface-cli upload` is a human-authorized final step per ROADMAP).
