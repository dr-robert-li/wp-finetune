    **Idempotency pattern (addresses review concern: operational reliability):**
    Every major step writes a completion marker file on success. On re-run, check marker before executing:
    ```python
    COMPLETION_MARKERS = {
        "profiling": "output/profiling/.complete",
        "eval_ratio": "output/eval_triage/ratio_{ratio}/.complete",
        "wpbench_ratio": "output/eval_triage/ratio_{ratio}/.wpbench_complete",
        "triage": "output/.triage_complete",
    }

    def step_complete(marker_path: str) -> bool:
        return Path(marker_path).exists()

    def mark_complete(marker_path: str):
        Path(marker_path).write_text(f"completed: {datetime.now().isoformat()}\n")
    ```
    Each step checks its marker first. If complete, log "Skipping {step} (already complete, marker: {path})" and continue. The `--force` CLI flag bypasses all markers for full re-run.
