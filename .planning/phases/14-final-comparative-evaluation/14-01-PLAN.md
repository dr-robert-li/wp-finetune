# Phase 14-01 — Final Comparative Evaluation (unpruned two-model pair)

**Phase:** 14 — Final Comparative Evaluation
**Depends on:** Phase 13 (closed `no_winner` -> ship unpruned)
**Requirements:** EVAL3-01, EVAL3-02

## Goal

Produce the final comparative eval report for the shipping stack. Because Phase 13 shipped no pruned
variant and Phase 10 shipped no RL baseline, both A/B arms are empty; the phase is a re-confirmation that
consolidates already-measured vLLM numbers with provenance and records the flat size line honestly.

## Approach decision

Do NOT re-run the vLLM wp-bench sweep. The shipping-stack numbers were measured under the identical
serving stack during Phase 11's k-sweep and are frozen in `output/sieve/optimal_k.json`. Re-running would
reproduce those figures at the cost of GB10 hours. Instead: consolidate, cite, and record the real
on-disk size measurement (which is new and cheap). This mirrors how Phase 13 closed PRUNE-06 by
documented disposition rather than an empty surgery run.

## Tasks

1. **Consolidate measured quality** into `output/eval3/eval3_final_comparison.json` (machine-readable) and
   `EVAL3-REPORT.md` (narrative). Copy wp-bench 0.4484, judge ensemble rho 0.8075, s1 0.8017 from
   `optimal_k.json` with the source path attached to each. [EVAL3-01]
2. **Record size + speed delta.** Measure bf16 on-disk size (`du -sh`, 57 GB/checkpoint), pull param
   counts from `config.json`, state the 0% size reduction and unchanged speed with the two prior
   pruning-dead findings as cause. [EVAL3-02]
3. **Document the vacuous A/B arms** with evidence pointers (Phase 13 no_winner, Phase 10 RL rejected).
4. **Confirm the wp-bench hard gate** for packaging: 0.4484 >= 0.4286 acceptance bar. [EVAL3-01 gate]
5. Close: SUMMARY, VERIFICATION, STATE + ROADMAP + REQUIREMENTS updates, atomic commit.

## Success criteria (goal-backward)

- wp-bench hard gate result recorded with pass/fail and source. ✅ 0.4484 >= 0.4286.
- Report covers 9-dim behavior, speed delta, size reduction, seed variance. ✅
- Every number traces to a named artifact. ✅
- No fabricated eval; consolidation is labeled as such. ✅

## Out of scope

Fresh GPU eval, per-dimension pruned-vs-baseline table (no pruned model exists), RL-baseline comparison
(no RL model shipped).
